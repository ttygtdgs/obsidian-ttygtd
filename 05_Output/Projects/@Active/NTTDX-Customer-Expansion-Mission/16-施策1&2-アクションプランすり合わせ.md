## 施策1: NTT東連携（インバウンド拡大）

### やりたいこと

NTT東の各部門に「AIチームの案件を紹介・共同提案してもらう」仕組みを作る。

### NTT東の組織構造

```
BI本部（ビジネスイノベーション本部）
├── ビジネス企画部 ← マーケット全体にサービスを展開する企画機能
├── 産業基盤ビジネス部 ← BI本部配下の現場組織。ロイヤルカスタマーを持つ
└── 支店（BI部） ← BI本部と縦構造だが、機能は近しく完全なピラミッドではない
```

※瀬谷さん補足: 支店のBI部はBI本部と縦の関係にあるが、機能的には近しい位置づけで完全な上下関係ではない。ビジネス企画部は産業基盤ビジネス部と同列のBI本部配下の個別部門。

### ターゲット4部門と役割

| 部門 | 位置づけ | 何をしてもらうか | 期待値 |
|------|---------|----------------|--------|
| **BI本部** | 統括組織 | AIサービスの全体方針の合意・承認 | 各部門への展開の起点 |
| **ビジネス企画部** | BI本部配下・企画機能 | AIサービスをNTT東の商材に載せてもらう（面的展開） | 中〜高単価案件を安定獲得 |
| **産業基盤ビジネス部** | BI本部配下・現場組織 | 既存ロイヤル顧客にAIを追加提案してもらう | 質の高い案件（自治体+民需） |
| **支店（BI部）** | BI本部と縦構造 | AI案件の紹介をしてもらう | 量。ただし立ち上げに時間 |

### 共通で準備するもの（3部門共通）

## 施策2: Web強化

### 現状

| 項目 | 状態 |
|------|------|
| AIチームのWebページ | **ほぼなし**（コーポレートサイトにサービス名の列挙のみ） |
| 競合比較 | 全指標で1〜1.5/5（サービス情報・事例・SEO・CTA全て不足） |
| 主要キーワードの検索順位 | **全て圏外**（「生成AI 導入支援」等） |
| 比較サイト掲載 | **ゼロ** |
| 全面更新予定 | **7月** |

### 方針案: 7月の全面更新に向けてできることから進める

| 優先 | やること | 概要 | 7月前にできるか |
|------|---------|------|:---:|
| **1** | サービスページの情報充実 | メニュー・価格帯・事例をページに載せる | △（更新が7月なら合わせて） |
| **2** | 事例コンテンツの準備 | 既存実績（茨城県AI、ケアマネAI等）を事例として整理 | ○ 素材の準備は可能 |
| **3** | 比較サイトへの掲載 | 起業LOG、AIsmiley等への掲載申請 | ○ すぐできる |
| **4** | 問い合わせCTAの改善 | 「資料DL」「無料相談」「見積依頼」の3段階化 | △ |
| **5** | コンテンツ制作体制 | ブログ・技術記事の制作ルーティン設計 | ○ 計画は立てられる |

### タイムライン（施策2）

```
3月〜4月 ───────────────────────────────────────
 ├── 事例コンテンツの素材準備（顧客許可確認含む）
 ├── 比較サイトへの掲載申請
 └── 7月更新に向けたページ構成案・原稿ドラフト

5月〜6月 ───────────────────────────────────────
 ├── ページデザイン・実装
 └── コンテンツ制作ルーティンの設計

7月 ────────────────────────────────────────
 └── サービスページ全面更新・公開
```
