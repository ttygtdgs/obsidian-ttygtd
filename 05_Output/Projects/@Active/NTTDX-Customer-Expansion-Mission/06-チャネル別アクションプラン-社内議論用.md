# チャネル別アクションプラン（社内議論用）

> **目的**: 各チャネルの攻め方を社内で議論・合意し、優先順位を決めて実行に移す。
> **前提**: サービスメニュー整理済み（[[01-施策1-サービスソリューション整理]]）、チャネル戦略設計済み（[[05-チャネル戦略-施策1&2深堀り]]）

## 【施策1】NTT東連携チャネル

---

### CH-A1. BI本部VP（面的展開）  ⭐優先度：高

**概要**
NTT東 BI本部VP（ビジネスイノベーション本部）は、マーケットに面的展開するサービス・仕組みの企画を担う部門。ここと連携し、AIサービスをNTT東の商材ラインナップに組み込む。

**ねらい**
- VP本部が自社顧客に**コセル（co-sell）**できる形でAIサービスを提供
- VP経由で安定的に中〜高単価案件の紹介を獲得する

**チャネル特性**

| リード量 | リード質 | 立ち上げ速度 | 運用工数 |
|---------|---------|------------|---------|
| 中 | 高 | 速い | 低 |

**主な流入フェーズ**: ①理解 → ②設計（研修パッケージ、活用設計が入口）

**アクションプラン**

| # | アクション | 成果物 | いつまで | 担当 | 状態 |
|---|----------|-------|---------|-----|------|
| 1 | VP本部との初回すり合わせ設定 | 面談日程 | | | 🔲 |
| 2 | 共同サービスメニュー案の作成 | NTT東既存サービス × AIサービスのセット提案書 | | | 🔲 |
| 3 | エグゼクティブサマリー（1枚）作成 | VP幹部向け：ROI・事業インパクト中心 | | | 🔲 |
| 4 | VP本部への正式プレゼン | プレゼン資料 | | | 🔲 |
| 5 | 定期パイプラインレビューの枠組み設計 | 四半期レビュー運用ルール | | | 🔲 |

**KPI目標**
- VP経由の紹介件数: 月2件以上
- 紹介→初回面談の転換率: 70%以上
- VP経由案件の受注金額: 四半期1,000万円以上

**📌 議論ポイント**
- [ ] VP本部の誰にアプローチするか？ファーストコンタクトの進め方
- [ ] 共同メニューに入れるサービスの優先順位（研修？PoC？）
- [ ] コセル時の利益配分・契約形態はどうするか

### CH-A2. 産業基盤ビジネス部（ロイヤルカスタマー経由）  ⭐優先度：高

**概要**
南支店BI（産業基盤ビジネス部）はロイヤルカスタマーを保有。既存の信頼関係がある顧客に対して、AIサービスをアドオン提案する。自治体向け・民需向けの両面。

**ねらい**
- **アカウントベースド戦略**: 上位10〜20社を特定し、個社別にAI活用仮説を準備
- 信頼関係のある顧客からスタートし、成功事例を横展開する

**チャネル特性**

| リード量 | リード質 | 立ち上げ速度 | 運用工数 |
|---------|---------|------------|---------|
| 低 | 非常に高 | 速い | 中 |

**主な流入フェーズ**: ②設計 → ③検証（ユースケース設計、PoCが入口）

**アクションプラン**

| # | アクション | 成果物 | いつまで | 担当 | 状態 |
|---|----------|-------|---------|-----|------|
| 1 | 産業基盤ビジネス部との初回面談設定 | 面談日程 | | | 🔲 |
| 2 | ターゲットアカウント選定（上位10〜20社） | ターゲットリスト | | | 🔲 |
| 3 | 自治体向けAI活用事例集の作成 | 事例集（住民サービス・内部効率化・データ活用） | | | 🔲 |
| 4 | 民需向け業種別AI活用仮説シートの作成 | 業種別仮説シート（製造・医療・農業・教育等） | | | 🔲 |
| 5 | 営業同行の仕組み設計 | 同行ルール・ヒアリングシート | | | 🔲 |
| 6 | PoC成功→横展開プレイブック作成 | 横展開手順書 | | | 🔲 |

**KPI目標**
- ターゲットアカウント接触数: 月3社以上
- 同行商談→案件化率: 30%以上

**📌 議論ポイント**
- [ ] 自治体 vs 民需、どちらを先に攻めるか？
- [ ] ターゲットアカウント選定の基準（規模？業種？既存関係の深さ？）
- [ ] 同行商談のリソース確保（誰が何件/月対応できるか）

### CH-A3. 支店ネットワーク（地域展開）  優先度：中

**概要**
NTT東の各地域支店の営業担当者がAI案件の"目利き"をできるようにし、紹介を増やす。「Train the Trainer」モデル。

**ねらい**
- 支店営業に「こういう相談が来たらAIチームに紹介」という判断基準を渡す
- パイロット支店で検証してから段階的に展開

**チャネル特性**

| リード量 | リード質 | 立ち上げ速度 | 運用工数 |
|---------|---------|------------|---------|
| 高 | 中 | 遅い（研修必要） | 高 |

**主な流入フェーズ**: ①理解（研修パッケージが入口）

**アクションプラン**

| # | アクション | 成果物 | いつまで | 担当 | 状態 |
|---|----------|-------|---------|-----|------|
| 1 | AI案件発見チェックリスト作成 | 1枚もの（顧客がこう言ったら→紹介） | | | 🔲 |
| 2 | 営業向けガイド作成（3〜5枚） | 紹介判断フロー付きガイド | | | 🔲 |
| 3 | 紹介フォーム整備 | Slack or Google Form | | | 🔲 |
| 4 | パイロット支店3拠点の選定・勉強会実施 | 勉強会資料（30分）、実施レポート | | | 🔲 |
| 5 | 紹介実績フィードバックの仕組み設計 | フィードバックテンプレート | | | 🔲 |
| 6 | 効果検証→全支店展開判断 | 検証結果、展開計画 | | | 🔲 |

**KPI目標**
- 活性化支店数（1件以上紹介した支店）: 30%以上
- 支店経由の紹介件数: 月5件以上
- 紹介→面談までのリードタイム: 5営業日以内

**📌 議論ポイント**
- [ ] パイロット支店3拠点はどこにするか？（選定基準）
- [ ] 勉強会の実施形態（オンライン？訪問？）
- [ ] 紹介フォームの仕組み（Slack? Google Form? 既存の仕組みに乗せる？）
- [ ] 支店展開にかかるリソースと、A1/A2との優先度バランス

## 【施策2】Webチャネル

---

### CH-B1. AIチームサービスページ  ⭐優先度：高

**概要**
AIチームの存在・サービスを伝え、問い合わせを獲得する「ホームベース」。現状はWebページが存在せず、オーガニックな問い合わせ導線がゼロ。

**ねらい**
- 「AIチームに相談したい」と思った人が、いつでもアクセスできる場所を作る
- 問い合わせ→初回面談の導線を自動化する

**ページ構成案**

| セクション | 内容 |
|-----------|------|
| ヒーロー | キャッチコピー＋提供価値を一言で |
| サービス概要 | 4パッケージ（研修/PoC/本番実装/伴走）を視覚的に |
| 導入事例 | 2〜3件をカード形式（業種・課題・成果） |
| チーム紹介 | メンバーの専門性・実績 |
| FAQ | 期間・費用感・セキュリティ等 |
| 問い合わせCTA | フォーム（選択式＋フリーテキスト） |

**問い合わせ対応フロー**

```
フォーム送信 → 自動返信メール → Slack通知 → 1営業日で担当アサイン → 3営業日で初回コール
```

**アクションプラン**

| # | アクション | 成果物 | いつまで | 担当 | 状態 |
|---|----------|-------|---------|-----|------|
| 1 | ページ構成・要件の確定 | 要件定義書 | | | 🔲 |
| 2 | 掲載コンテンツの原稿作成 | 各セクションのテキスト・画像素材 | | | 🔲 |
| 3 | 問い合わせフォーム設計 | フォーム項目定義、対応フロー | | | 🔲 |
| 4 | デザイン・実装（社内Web担当 or 外注） | 公開可能なページ | | | 🔲 |
| 5 | Slack通知・自動返信メール設定 | 通知設定完了 | | | 🔲 |
| 6 | 広報・Web担当部門との調整・承認 | 公開許可 | | | 🔲 |
| 7 | ページ公開 | 公開URL | | | 🔲 |

**KPI目標（公開6ヶ月後）**
- ページ訪問数: 月500PV以上
- 問い合わせ送信数: 月3件以上
- 問い合わせ→面談設定率: 80%以上

**📌 議論ポイント**
- [ ] ページはどこに作るか？（DXPNコーポレートサイト内？独立ページ？）
- [ ] 制作は社内？外注？
- [ ] 掲載する事例は許可取れるものがあるか？（匿名可？）
- [ ] 広報部門の承認プロセスはどうなっているか？

### CH-B3. セミナー・ウェビナー  優先度：中

**概要**
定期的なセミナーでリードを獲得し、参加者を個別相談→案件化につなげる。

**ねらい**
- 「広く浅く」の認知獲得と、「狭く深く」の関係構築を使い分ける
- セミナー資料が支店向け勉強会素材にも再利用できる（チャネル間連携）

**セミナー体系**

| ハンズオンWS | 対面 or オンライン半日 | 四半期1回 | 深い関係構築 |
| NTT東共催セミナー | 対面半日 | 半期1回 | NTT東顧客リード |

**アクションプラン**

|---|----------|-------|---------|-----|------|
| 1 | 初回ウェビナーのテーマ・日程決定 | 企画書 | | | 🔲 |
| 2 | セミナー資料の作成 | スライド | | | 🔲 |
| 3 | 告知・集客の仕組み設計 | 告知チャネル・登録フォーム | | | 🔲 |
| 4 | 参加者フォローアップフロー設計 | アンケート・フォローメールテンプレ | | | 🔲 |

## 【連携】チャネル間フライホイール

施策1と施策2を独立に動かすのではなく、**相互に燃料を供給し合う**仕組み。

| パターン | 内容 | 効果 |
|---------|------|------|
| **Web → NTT東コセル** | Webからの問い合わせ顧客がNTT東テリトリー内なら共同対応 | 成約率UP |
| **NTT東紹介 → Web活用** | 紹介案件の初回面談前に関連コンテンツを事前共有 | 商談の質向上 |
| **セミナー → 支店勉強会** | ウェビナー録画・スライドを支店向け素材に再利用 | コンテンツ一石二鳥 |
| **事例フライホイール** | 全チャネルの成功案件→事例化→全チャネルに展開 | 全体加速 |

> **最も重要な仕組み**: 事例が増えるほど全チャネルのパフォーマンスが上がる循環を作ること。
