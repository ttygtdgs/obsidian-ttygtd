## 施策1: インバウンド拡大（NTT東連携強化）

### ゴール
NTT東の各部門・支店に対して、AIチームの取り組みを効果的に紹介し、案件紹介を増やす。

### 方針
1. 分かりやすいプロダクトパッケージを準備する
2. そのプロダクトパッケージの紹介資料を準備する
3. 資料をNTT東に展開する（メール/Teamsおよびミーティング）
   1. 本社
      1. BI本部VP：面的展開（マーケットに対して展開したい、サービスや仕組み作りに関心がある）
      2. 南支店BI（産業基盤ビジネス部）：ロイヤルカスタマーを持っている
   2. 支店（）

> 自治体は両方やっている。自治体向けの民需向けの用意が必要になりそう

### タスク一覧

#### Phase 1: プロダクトパッケージの整理

##### 1.3 サービス案レビュー
- [ ] 適切な承認フローにて実施

#### Phase 2: 紹介資料の作成

## 施策3: 公募案件取得

### ゴール
公募案件を定期的にキャッチし、適切な案件に迅速に応募できる体制を構築する。

### 方針
1. 今年度の公募案件をベースに今後手を挙げる案件の基準を明確化し、共通認識を図る
2. ニュースを定期取得するワークフローを構築する
3. 週1〜隔週で案件共有・申請判断の会議体を設定する

### タスク一覧

#### Phase 1: 現状整理・基準策定

##### 1.1 今年度の公募案件整理
- [ ] 今年度の主要公募案件の洗い出し

##### 1.2 案件選定基準の策定
- [ ] 「手を挙げる案件」の基準を明確化
  - [ ] 対象領域（AI、DX、データ活用等）
  - [ ] 予算規模（下限・上限）
  - [ ] 期間・工数
  - [ ] 必要なリソース・体制
  - [ ] 戦略的重要性
- [ ] 「手を挙げない案件」の基準を明確化
- [ ] 基準のドキュメント化

##### 1.3 チーム内共通認識形成
- [ ] 選定基準案のチーム内共有
- [ ] ディスカッション・調整
- [ ] 最終版の合意・承認

#### Phase 2: ワークフロー構築

##### 2.1 情報収集の自動化
- [ ] 情報収集対象サイトの確定
- [ ] 自動収集ツールの選定・構築（n8n、RSS、Google Workspace/Studio等）
- [ ] 収集条件（キーワード、カテゴリ等）の設定
- [ ] 収集テスト・調整

##### 2.2 情報集約・通知
- [ ] 収集情報の集約先の決定（Slack/スプレッドシート等）
- [ ] 通知フォーマットの設計
- [ ] 通知頻度の設定（日次/週次）
- [ ] ワークフローの本番稼働

#### Phase 3: 会議体設定
- [ ] 会議の目的・アジェンダの定義
- [ ] 開催頻度の決定（週1 or 隔週）
- [ ] 参加者の決定
- [ ] 事前準備ルール（案件リスト共有タイミング等）
- [ ] 議事録・決定事項の記録方法
- [ ] 初回会議の日程調整
- [ ] 定例会議の設定・運用開始

## 施策4: IT導入補助金（デジタル化・AI導入補助金）

### ゴール
IT導入補助金を活用した案件獲得の仕組みを整理し、顧客への提案力を強化する。(2026年度からデジタル化・AI導入補助金)

### 方針
Dify・n8nが今年度ITツールとして登録されていることを活用し、以下を整理する：

1. IT導入補助金を活用できる案件パターンの整理
2. 各パターンでの利益率・収益性の試算
3. 顧客への提案方法の確立

### タスク一覧

#### Phase 1: 現状把握
- [ ] 今年度および来年度以降のIT導入補助金制度の概要確認
  - [ ] 補助対象
  - [ ] 補助率・上限額
  - [ ] 申請要件
  - [ ] スケジュール（申請締切等）
- [ ] Dify・n8nの登録状況の確認
  - [ ] 対象となる補助枠
  - [ ] 価格設定
- [ ] 競合調査（今年度上記ツールを支援している事業者の提案を調査する）

#### Phase 2: 案件パターン整理
- [ ] Dify・n8nで実現できるソリューションの整理
- [ ] 各ソリューションの標準構成・価格帯の整理
- [ ] 補助金適用時の顧客負担額の試算

#### Phase 3: 収益性分析
- [ ] パターン別の売上・コスト試算、利益率の算出
- [ ] 最低受注金額の設定
- [ ] 推奨案件規模の設定

#### Phase 4: 提案体制構築
- [ ] IT導入補助金活用提案書テンプレート作成
- [ ] 顧客向け説明資料（補助金の仕組み説明）
- [ ] 費用シミュレーション資料
- [ ] 補助金活用案件の提案フロー設計
- [ ] チーム内への展開・勉強会実施
