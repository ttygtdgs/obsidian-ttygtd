## 個別案件詳細

### 1. 大阪府 — 生成AI庁内利活用推進事業

- **落札者**: 富士通Japan
- **金額**: 9,293万円（税込）
- **案件概要**: 大阪府庁の全職員を対象とした生成AI利活用環境の構築・運用。2023年11月〜2024年1月に庁内全職員への試験導入を経て、本格的な利活用推進事業として発注
- **実装要件**:
  - 庁内専用の生成AI利用環境構築
  - 職員向け研修・ガイドライン策定
  - セキュリティポリシーに準拠した環境設計
  - 富士通の「Takane」LLM（日本語特化）やAzure OpenAI Serviceの活用が想定
- **実装規模**: 大阪府庁全職員（約8,000〜9,000名規模）
- **セキュリティ要件**: 入力データの非学習利用、庁内専用環境での運用
- **契約期間**: 2024年度〜（詳細非公開）
- **落札の背景**: 富士通Japanは自治体向けシステム基盤に強く、富士通独自のLLM「Takane」やエンタープライズAIフレームワークを保有。大阪府との既存取引関係も影響
- **NTTDXPNフェーズ**: ④実装（構築）+ ⑤定着（運用）

- **実装規模**: 東京都公式サイト利用者全体（住民向け）

- **契約期間**: 2025年3月〜（令和7年度）

### 3. 山口県 — 生成AI・学習アシスタントアプリケーション運用保守

- **落札者**: スタディポケット株式会社
- **金額**: 4,356万円（税込）
- **案件概要**: 山口県教育委員会が推進する「生成AIを活用した家庭と学校の学びの好循環創出事業」。県内全公立中学校153校に生成AIサービスを導入（都道府県単位で全国初）
- **実装要件**:
  - 「StudyPocket for STUDENT」の提供
  - Azure OpenAI Serviceベースの生成AI
  - コンテンツフィルタリング機能（暴力・ヘイト・性的・自傷の4カテゴリー）
  - マルチモーダル対応（OCR、画像分析、多言語音声入出力）
  - 探究学習モード
  - Machine learning opt-out（ユーザー入力がモデル学習に使用されない）
- **実装規模**:
  - 初期: 6市町村7校 約1,500名（2024年6月〜）
  - 拡大: 全県153校 生徒・教職員 約33,000名（2025年5月〜）
- **セキュリティ要件**: ニックネーム利用、SSO（Google/Microsoft）、個人情報第三者非提供、国内データセンター
- **契約期間**: 2024年6月〜2025年3月（初期）、2025年5月〜（全県展開）
- **落札の背景**: スタディポケットは教育特化型生成AIサービスのパイオニア。文部科学省の生成AIパイロット事業での実績
- **NTTDXPNフェーズ**: ④実装（構築）→ ⑤定着（運用）

- **落札の背景**: ソフトバンクは自治体向け生成AIソリューションを展開。Azure OpenAI Serviceの導入支援実績あり
- **NTTDXPNフェーズ**: ③検証（PoC）

## トレンド分析

### LLMトレンド

| LLM/サービス | 採用案件数 | 案件例 |
|-------------|----------|--------|
| Azure OpenAI Service (GPT-4系) | 8件以上 | 大阪府、山口県、渋谷区、葛飾区、山梨県、奈良県、千葉県、北海道 |
| Microsoft Copilot | 2件 | 渋谷区、北海道（無償版） |
| Claude 3 Opus (Anthropic) | 1件 | 千代田区（マルチLLM） |
| Gemini Pro (Google) | 1件 | 千代田区（マルチLLM） |
| 富士通 Takane | 1件（想定）| 大阪府 |
| NEC cotomi | 1件（想定）| 東京都 |

**分析**: Azure OpenAI Serviceが圧倒的シェア。理由は以下の3点:
1. マイクロソフトのエンタープライズセキュリティ（入力データ非学習保証）
2. 自治体で既に導入済みのMicrosoft 365/Teamsとの親和性
3. ISMAP（政府情報システムのためのセキュリティ評価制度）認定

### アーキテクチャトレンド

| 技術要素 | 採用件数 | 備考 |
|---------|---------|------|
| RAG（検索拡張生成） | 8件以上 | ほぼ全ての庁内利活用案件で必須 |
| クラウド（Azure） | 10件以上 | Azure優勢、オンプレ要件は少数 |
| マルチLLM対応 | 1件 | 千代田区のみ（教育分野で先進的） |
| AIエージェント | 2件 | 東京都（NEC）、葛飾区（NTT-AT、試験運用） |
| チャットボット/アバター | 3件 | 東京都、今治市、千葉県 |

**分析**:
- RAGが自治体生成AIの「標準装備」として定着。庁内文書・マニュアル・議事録等の独自データをLLMに接続する需要が極めて高い
- クラウドはAzure一強。AWS・GCPの採用は確認されず
- AIエージェントは2025年後半から登場し始めた最新トレンド（次世代案件の主軸になる可能性）

### セキュリティトレンド

| セキュリティ要件 | 採用状況 |
|----------------|---------|
| 入力データの非学習利用保証 | ほぼ全件 |
| 閉域網接続（SD-WAN等） | 葛飾区、千葉県 |
| LGWAN接続 | 北海道（非想定）、NTT東日本（オプション提供） |
| コンテンツフィルタリング | 教育分野で必須（山口県、千代田区） |
| 個人情報マスキング | 広島県 |

**分析**:
- LGWAN接続は「必須」ではなく「オプション」の位置づけが多い。インターネット経由＋閉域網接続が主流
- 教育分野ではコンテンツフィルタリングが必須要件として定着
- 入力データの非学習保証はAzure OpenAI Serviceのデフォルト機能で対応するケースが大半

## NTTDXPNへの示唆

### NTTグループの強み
1. **ネットワーク＋AI一体型提案**: 閉域網接続（SD-WAN）とAIプラットフォームを一括提供できる唯一のプレーヤー
2. **地域密着営業**: NTT東日本の支店ネットワークによる自治体との既存関係
3. **自社プロダクト**: 「ミンクスプラス生成AI」による標準化された提案が可能
4. **グループシナジー**: NTT東日本（ネットワーク）+ NTT-AT（AI実装）+ NTTデータ（大規模SI）

### 参入余地のある領域
1. **AIエージェント案件**: 2025年後半から本格化。NEC「cotomi Act」が先行しているが、市場はまだ形成初期
2. **教育分野**: スタディポケット・アルサーガが先行するも、NTTグループは渋谷区実績を横展開可能
3. **住民向けフロントエンドサービス**: チャットボット・移住相談等の住民接点系はNTTの通信インフラとの相性が良い
4. **PoC→構築のアップセル**: 効果検証（1,800〜2,200万円）から構築（2,500〜5,000万円）へのステップアップ提案

### 競合への対抗策
- **富士通/NEC**: 独自LLM（Takane/cotomi）を持つが、NTTは「ネットワーク＋AI」のトータルソリューションで差別化
- **コンサル系（アビーム等）**: 上流設計から入り込まれるリスク。NTTも設計フェーズからの提案力強化が必要
- **ベンチャー系（スタディポケット等）**: 特化型SaaSに対しては、汎用プラットフォーム＋カスタマイズで対抗
