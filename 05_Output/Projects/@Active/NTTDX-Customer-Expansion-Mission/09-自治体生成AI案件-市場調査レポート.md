# 自治体における生成AI案件 — 市場調査レポート

> **目的**: NTTDXパートナーとして自治体向け生成AI案件の提案創出に向けた市場調査
> **調査日**: 2026年2月17日
> **データソース**: 入札王データ（887件）、総務省統計、デジタル庁資料、Web調査

---

## エグゼクティブサマリー

自治体の生成AI市場は**2023年から2025年にかけて爆発的に成長**し、入札案件数は年間13件→91件→124件と推移。累計落札金額は**37.4億円**に達している。2026年度はデジタル庁の「ガバメントAI」自治体展開、DS-920ガイドライン全面適用、第2世代交付金2,000億円の追い風により、**さらなる市場拡大が確実**。

NTTDXパートナーにとっての最大の機会は：
1. **約半数の案件が1社応札**（48.7%） → 応札するだけで勝てる可能性
2. **Medium〜Large帯（500万〜1億円）に93件/年** → 技術力で差別化可能
3. **RAG案件が毎年3倍増** → NTTDXPNの技術的強みと合致
4. **市区町村の70%がまだ未導入** → 巨大な潜在需要

### 4.3 課題・障壁

| 課題 | 詳細 |
|------|------|
| **ハルシネーション** | 自治体職員の最大の懸念。行政の正確性要求と相反 |
| **個人情報保護** | 入力データのAI学習利用への懸念。LGWAN環境が解決策に |
| **職員の利用率** | 導入しても「使われない」問題（下記詳細）。研修・定着支援が鍵 |
| **予算確保の壁** | 小規模自治体の56%が「予算化できるか不明」 |
| **セキュリティ要件** | α'モデル準拠、LGWAN接続、個人情報の取扱い |

### 4.4 最大の課題: 職員の利用率が上がらない問題 → NTTDXPNの商機

多くの自治体が「導入したが使われない」問題に直面しており、**これこそがNTTDXPNの提案機会**。

| 自治体 | 状況 |
|--------|------|
| **東京都** | 5万人が利用可能 → 4か月で申請はわずか**1割（5,000人）** |
| **滋賀県** | 試行利用者の**76.2%が最初の1回のみ**で離脱 |
| **横須賀市** | 導入直後の盛り上がりの後低迷 → **1年かけて**ようやく60%に回復 |

**有効な対策（提案に使える知見）**:
- 「ガイドライン整備 → 小規模検証 → 全庁展開」の3段階アプローチ
- 専門家の伴走支援で**70%が実業務利用**に到達
- プロンプト不要の「モード機能」（福岡市QT-GenAI方式）
- 業務別プロンプトテンプレートの提供
- 庁内報での具体的活用事例の共有（横須賀市の成功策）

**→ NTTDXPNのPhase①研修 + Phase⑤定着支援が直接的な解決策になる**

### 5.2 ガバメントAI「源内」の詳細

デジタル庁が内製開発した行政向け生成AI利用環境。

| 項目 | 内容 |
|------|------|
| 基盤 | AWS GenU（オープンソース）をベースに行政特化拡張 |
| 機能 | 20種類以上のAIアプリ（業務マニュアル検索、国会答弁検索等） |
| 展開 | 2026年1月: 試験利用開始、5月: 他府省庁展開、夏: 国産LLM導入 |

| 国産LLM | 公募実施（2025年12月〜2026年1月）、27年度以降本格提供 |

**NTTDXPNへの示唆**:
- 源内は「基盤」であり、**自治体固有の業務への適用・カスタマイズ・運用支援**に商機
- 源内では対応しきれない**RAG構築・データ連携・業務特化AI**がNTTDXPNの出番
- 小規模自治体への**導入支援・研修**も重要な役割

### 6.2 有望な提案テーマ（自治体ニーズ別）

#### テーマ1: 庁内生成AI導入パッケージ ★★★

**市場規模**: 年間約100件（実装+運用）、Medium帯中心
**背景**: 市区町村の70%がまだ本格導入していない。源内の自治体展開を待てない/待つ間のつなぎ需要。

**提案内容**:
- LGWAN対応の生成AI環境構築（Dify/Azure OpenAI）
- 業務別プロンプトテンプレート整備
- 職員向け研修（基礎+実践）
- 運用サポート（月額）

**想定単価**: 導入300〜800万円 + 運用月額20〜50万円
**差別化**: Dify/n8nによるカスタマイズ性 + NTTグループの信頼性

#### テーマ2: RAGシステム構築 ★★★

**市場規模**: 年間10〜15件（急成長中、毎年3倍ペース）
**背景**: 庁内文書・条例・マニュアルの検索需要。源内では対応しきれない自治体固有のナレッジ管理。

**提案内容**:
- 条例・規則・マニュアルのRAG構築
- 住民向けFAQ自動生成
- 議事録・答弁検索システム

**想定単価**: 500〜2,000万円
**差別化**: RAG構築の技術力（Dify+独自プロンプト設計）

#### テーマ3: 住民向けAIチャットボット ★★☆

**市場規模**: 年間40〜50件（成熟市場だが安定需要）
**背景**: 従来型チャットボットから生成AI対応へのリプレース需要。

**提案内容**:
- 生成AI搭載の次世代チャットボット
- 多言語対応（インバウンド対策）
- 窓口業務のAI化（電話応対、ボイスボット）

**想定単価**: 200〜500万円（導入）+ 月額10〜30万円（運用）
**注意**: サイネックス・ビースポーク等の既存SaaS勢が強い。差別化必須。

#### テーマ4: 生成AI研修・ワークショップ ★★☆

**市場規模**: 年間40件、大型は4,800万円まで
**背景**: AI法施行により自治体の「AI活用」が責務に。職員のリテラシー向上が急務。

**提案内容**:
- 経営層向けAI戦略セミナー
- 現場職員向けプロンプト実践研修
- DX推進担当者向けPoC設計ワークショップ

**想定単価**: 30〜80万円（研修）、100〜200万円（シリーズ研修）
**差別化**: 実際のPoC/実装につなげる一気通貫の提案力

#### テーマ5: AI活用ロードマップ策定支援 ★★☆

**市場規模**: 少数だが高単価
**背景**: DS-920ガイドラインでCAIO設置義務化。計画的なAI導入が求められる。

**提案内容**:
- 現状業務のAI適用性診断
