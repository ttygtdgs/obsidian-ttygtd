# 長野県 生成AI公募 分析メモ

> **ステータス**: 社内確認中
> **対応期限**: 公募締切に合わせて要確認
> **担当**: NTTDXPN

---

## 1. 公募概要

| 項目 | 内容 |
|---|---|
| **業務名** | 令和8年度 生成AIによる業務効率化推進事業業務 |
| **発注者** | 長野県 企画振興部 DX推進課 |
| **予算上限** | 29,762,000円（税込） |
| **契約期間** | 契約日〜令和9年3月31日 |
| **利用者数** | 職員 約8,000名（同時接続 最大400名） |
| **トークン量** | 1人月5万トークン（入力:出力＝9:1） |

---

## 2. 必須技術要件

| # | 要件 | 重要度 |
|---|---|---|
| 1 | **Azure OpenAI** ベース（仕様書に明記） | 必須 |
| 2 | **GPT-5相当以上** のモデル | 必須 |
| 3 | **LGWAN** 対応（24/365 平日休日問わず） | 必須 |
| 4 | **EntraID** 連携認証 | 必須 |
| 5 | **Teams** インターフェース | 望ましい |
| 6 | **RAG機能**: 160GB以上、5,000ファイル以上、業務別領域10程度 | 必須 |
| 7 | **国内データセンター**、日本の裁判管轄 | 必須 |
| 8 | **ISMAP等** セキュリティ評価制度の登録 | 必須 |
| 9 | データがOpenAI学習に使われない保証（オプトアウト） | 必須 |
| 10 | Web検索機能併用可 | 必須 |

### 要件の読み解き

長野県が求めているのは **Azure OpenAIの「エンジン」ではなく「完成車」**（SaaS）。

```
┌──────────────────────────────────────────────┐
│  長野県が欲しいもの（完成品SaaS）             │
│  ┌────────────────────────────────────┐      │
│  │ チャットUI / RAG / 管理画面 /      │      │
│  │ テンプレート / ユーザー管理 /      │      │
│  │ ログ・統計 / Web検索 ...          │      │
│  │  ┌──────────────────────────┐     │      │
│  │  │  Azure OpenAI Service   │ ←基盤│      │
│  │  │  (GPT-5等のLLM API)     │      │      │
│  │  └──────────────────────────┘     │      │
│  └────────────────────────────────────┘      │
│  + LGWAN接続 / EntraID認証 / 研修 / 運用     │
└──────────────────────────────────────────────┘
```

### Azure OpenAI単体 vs. 公募の要件

| | Azure OpenAI単体 | 公募の要件 |
|---|---|---|
| LLM（GPT-5等） | ✅ APIで呼べる | ✅ 必要 |
| チャットUI | ❌ APIのみ | ✅ 職員が使えるWebUI |
| RAG機能 | ❌ 自分で構築が必要 | ✅ 160GB+、領域管理 |
| ユーザー管理 | ❌ なし | ✅ 8,000名管理 |
| プロンプトテンプレート | ❌ なし | ✅ 自治体向けプリセット |
| 利用統計ダッシュボード | ❌ なし | ✅ トークン数・利用状況 |
| Web検索連携 | ❌ 自分で構築 | ✅ 最新情報を参照 |
| LGWAN接続 | ❌ 自分で構築 | ✅ 必須 |
| EntraID認証 | △ Azure ADと統合可 | ✅ 必須 |
| ファイル添付 | ❌ 自分で構築 | ✅ PDF/Office等 |

→ **既製品SaaSの再販がベストアプローチ**

## 4. 提案候補商材の比較

### Tier 1: 主要候補

#### (A) ミンクスプラス生成AI（NTT東日本）← **本命**

| 要件 | 適合 | 詳細 |
|---|---|---|
| Azure基盤 | ✅ | Microsoft Azure最新モデル提供 |
| LGWAN | ✅ | 接続オプション（2025年9月〜） |
| EntraID | ✅ | 必須要件として対応 |
| RAG | ✅ | 拡張ドキュメント検索機能搭載 |
| テンプレート | ✅ | 100以上のプロンプトテンプレート |
| GPT-5相当 | ❓ | 「最新言語モデル」提供と記載。R8年度にはGPT-5提供の可能性大 |
| Teams連携 | ❓ | 公開情報なし。要確認 |
| 160GB+ RAG | ❓ | 5GB単位で追加可能。上限は要確認 |
| ISMAP | ❓ | 公開情報なし。要確認 |
| Web検索 | ❓ | 公開情報なし |
| 8,000名対応 | ❓ | 月額14万円〜/20ID。大規模向け料金体系は要確認 |

- **料金**: 初期10万円 + 月額14万円〜（20ID〜）
- **対応ファイル**: PDF、Word、PowerPoint、Excel、JSON、HTML
- **NTTグループ内商材** → 調達・パートナーシップで有利
- 長野県はNTT東日本エリア → 地域密着サポート可能

#### (B) NTTスマートコネクト 生成AIサービス

| 要件 | 適合 | 詳細 |
|---|---|---|
| Azure基盤 | ✅ | Azure OpenAI Service明記 |
| LGWAN | ✅ | 対応（オプション） |
| RAG | ✅ | **2GB〜無制限**（カスタマイズ可）← 160GB+対応可能 |
| GPT-5相当 | △ | 現状GPT-4o。R8年度には更新見込み |
| EntraID | ❓ | 2段階認証あり。EntraIDは要確認 |
| Teams連携 | ❓ | 公開情報なし |
| ISMAP | ❓ | 公開情報なし |

- **料金**: 初期6.9万円 + 月額4.25万〜35万円（**同時接続数ベース** ← 長野の要件に合致）
- RAG容量が無制限カスタマイズ可能 → 160GB+対応の確度高い

#### (C) GaiXer（FIXER社）← **ISMAP最強**

- **ISMAP-LIU登録済**（2024年12月）← 最大の強み
- **LGWAN-ASP登録済**（2025年1月）
- Azure OpenAIベース
- 自治体・中央省庁の実績あり
- NTTDXPNとして再販・SIパートナーになれる可能性

#### (D) exaBase 生成AI for 自治体（ExaWizards社）← **長野案件には不適合**

- **47都道府県中25県庁**で利用実績（2026年2月現在、200以上の官公庁・自治体）
- RAG機能標準搭載（Word/Excel/PDF対応）
- 複数LLM対応（ChatGPT, Claude, Gemini）追加料金なし
- 自治体特化のUIとプロンプトテンプレート
- AIエージェント・Deep Research機能搭載
- 料金: 月額6.5万円〜（同時接続数ベース、アカウント無制限）
- LGWAN対応済み
- **⚠️ ISMAP未対応**（対応準備中と公式記載）
- **⚠️ 基盤はAWS東京リージョン**（Azure OpenAIではない）→ 仕様書の「Azure OpenAI」要件を満たさない
- → **本案件では候補外**。ただし他の自治体案件では有力候補

### Tier 2: 代替候補

## 5. NTTDXPNとしての提案戦略

### 推奨パターン: SaaS再販 + SI付加価値

```
[ミンクスプラス or GaiXer] ← SaaSライセンス（主商材）
        +
[NTTDXPNのSI] ← RAG構築・チューニング・研修・運用支援（付加価値）
```

### 提案の選択肢

| 選択肢 | 内容 | 評価 |
|---|---|---|
| **A: 既製品SaaS再販（推奨）** | ミンクスプラス等を主商材に、研修・RAG構築・運用をSIとして付加 | ✅ 開発不要、実績あり、工期短い |
| B: Azure OpenAIで自前構築 | Azure OpenAI + Dify/n8n等で独自システム構築 | ❌ 開発コスト大、ISMAP/LGWAN認証取得困難、予算3,000万では厳しい |

→ **選択肢Aが現実解**

### 予算配分イメージ（約3,000万円）

| 費目 | 金額 |
|---|---|
| SaaSライセンス | 1,200〜1,500万円/年 |
| RAG環境構築・ナレッジ整備 | 500〜800万円 |
| 初期導入研修・利用促進 | 200〜300万円 |
| 運用保守・効果検証 | 300〜500万円 |
| PM・進捗管理 | 200〜300万円 |

### 第一候補: ミンクスプラス生成AI（NTT東日本）

**推す理由**:
1. **NTTグループ内商材** → 調達・パートナーシップで有利
2. **EntraID対応済み** ← 仕様書の必須要件
3. **LGWAN対応済み** ← 仕様書の必須要件
4. **Azure基盤** ← 仕様書指定
5. 長野県はNTT東日本エリア → 地域密着サポートが可能

### 代替案: ハイブリッド構成

ミンクスプラス単体で全要件カバーできない場合:

```
ミンクスプラス生成AI（NTT東日本） ← チャット基盤・認証・管理
  + GaiXer（FIXER） ← ISMAP-LIU・高度RAG（要件充足の保険）
  + NTTDXPNのSI ← RAG構築・研修・運用
```

### NTTDXPNの強みを活かせるポイント

| ポイント | 内容 |
|---|---|
| **研修・利用促進** | AI研修のノウハウ（既存サービスメニュー20-80万円） |
| **RAGチューニング** | Dify/n8nでのRAG構築経験 |
| **効果検証** | 定量的評価指標の設計・測定 |
| **月次定例** | PM・進捗管理体制 |
