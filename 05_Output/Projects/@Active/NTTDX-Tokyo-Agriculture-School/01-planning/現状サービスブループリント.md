# 東京都農業高校-資格補助案件 現状サービスブループリント（As-Is）

**目的**: 既存システム／運用の流れと関係者の役割を可視化し、課題抽出と改善検討の土台とする。

## フロー全体像

1. 情報収集・問い合わせ
2. 申請準備・提出
3. 審査・実績報告

## ステークホルダーと役割

| 関係者 | 主な役割 | 現状の課題 |
| --- | --- | --- |
| 高校生・保護者 | 申請・証憑提出・支払申請 | 申請要件が複雑で不備率が高い |
| ネクシア（BPO） | 申請受付、問い合わせ対応、審査補助、支払データ作成 | 手作業・Excel依存で担当者負荷が高い |
| DXPN 教育事業部 | PM、自治体窓口、提案・入札準備 | システム改善余地の把握に時間を要する |
| DXPN SD事業部 | Web/システム保守・軽微改修 | 既存システムの仕様がR7踏襲でドキュメント不足 |
| 東京都（自治体） | 予算・要件提示、最終承認 | 詳細な課題はこれからヒアリング予定 |

## 現状ブループリントから見える主な課題

1. **問い合わせ対応の属人化**: FAQが紙・Excelに散在し、ネクシア窓口に負荷集中。
2. **申請・証憑チェックの手作業**: バリデーション不足により差し戻し多発。証憑確認も目視で非効率。
3. **口座確認・支払データ作成のリスク**: 口座情報の突合が人依存で、誤振込リスクを内包。
4. **データ連携・可視化が不十分**: 申請状況や問い合わせ履歴が統合されておらず、全体状況を俯瞰しづらい。
5. **ナレッジ更新が後手**: マニュアル更新や報告資料作成も手作業で、改善サイクルが回りにくい。
