# R8年度 都立農業系高校生資格取得支援制度
## 補助金支給事務委託 - 提案準備資料

**作成日**: 2026-02-10
**案件名**: 令和8年度「都立農業系高校生資格取得支援制度」の補助金支給に係る申請受付・審査業務委託
**委託者**: 東京都教育庁都立学校教育部高等学校教育課
**契約期間**: 令和8年4月1日〜令和9年3月31日
**R7落札実績**: 日本ビジネスクリエイト **1,245万円**

---

## 1. 案件分析

### 1.1 委託内容の全体像

本案件は **BPO業務（事務局運営）+ Webシステム構築・運用** の一体型委託。

| 領域 | 内容 | 座組（想定） |
|------|------|------------|
| PM・入札対応 | フロント、自治体窓口 | DXPN教育事業部 |
| BPO業務 | 事務局運営、申請審査、問合せ対応 | ネクシア |
| Webシステム | 申請フォーム構築・運用・保守 | DXPN SD事業部 |

### 1.2 受託者要件

- **プライバシーマーク** or **ISMS認証（ISO/IEC 27001）** の取得必須
- 業務は全て **日本国内** で実施
- lg.jpドメインでのWeb運用

### 1.3 R7→R8 仕様変更の主要ポイント

R7仕様書が手元にないため確定比較は不可だが、既存分析メモから：
- 別紙1の内容は**R7踏襲**の可能性が高い
- 申請件数: 約 **2,300件**（R7同等と推定）
- 対象資格: 約 **30種類**

### 1.4 競合分析

| 項目 | 内容 |
|------|------|
| R7受注者 | 日本ビジネスクリエイト |
| R7落札額 | 1,245万円（税込推定） |
| 入札形式 | 一般競争入札（推定） |
| 想定入札時期 | R8年2〜3月頃 |

**考察**: 1,245万円でBPO+システム一式を12ヶ月運用。かなりタイトな予算。

## 3. システムアーキテクチャ

### 3.1 アーキテクチャ比較

| 項目 | A: フルスクラッチ開発 | B: 既存改修型 | C: SaaS活用型 |
|------|-------------------|-------------|--------------|
| 概要 | Next.js + PostgreSQL on AWS | R7システムを引継ぎ改修 | kintone/Salesforce等を基盤に構築 |
| 初期費用 | 400〜560万円 | 200万円 | 150〜300万円 |
| 構築期間 | 1ヶ月（入札前に先行設計） | 2ヶ月 | 1〜2ヶ月 |
| 5月稼働 | **○ 先行着手で対応可能** | △ 引継ぎ前提 | △ 未経験SaaS習得リスク |
| lg.jp対応 | **○ 自由設計** | ○ 既存踏襲 | △ 要カスタムドメイン設定 |
| セキュリティ要件 | **○ AWS基盤で全要件対応** | ○ 既存基盤流用 | △ SaaS側制約あり |
| ランニング | **インフラのみで安い（約2万円/月）** | 中程度 | SaaSライセンス高め |
| 保守・拡張性 | **○ フルコントロール** | △ 既存設計に依存 | △ SaaS制約内 |
| リスク | 開発遅延リスク（先行着手で軽減） | 既存コード品質依存 | SaaS制約による要件不適合 |

### 3.2 推奨: A案（フルスクラッチ on AWS）

**理由**:
1. 案件規模が小さく（年間2,300件、管理者数名）、SaaS基盤を入れるメリットが薄い
2. システムの本質は**フォーム＋ステータス管理＋ファイルアップロード**であり、Next.js + Prisma + PostgreSQLで十分に対応可能
3. SaaS（kintone等）を挟むと、認証・ファイル・メールは結局自前実装になり、APIの間接層が増えるだけで複雑化する
4. 既存の開発スタック（Next.js + TypeScript）の経験を活かし、学習コストなしで構築可能
5. 入札前に設計・プロトタイプを先行着手することで、4月契約→5月稼働のタイムラインに対応

#### 構成

```
[申請者（スマホ/PC）]
    │
    ▼
[CloudFront Pro + WAF] ← lg.jpドメイン（都側DNS設定）
    │
    ▼
[Next.js フロントエンド] ← ECS Fargate / Vercel
    │ （申請フォーム、マイページ）
    ▼
[API Gateway / Express.js バックエンド] ← ECS Fargate
    │
    ├── [RDS PostgreSQL] ← 申請データ、ユーザーデータ
    ├── [S3] ← 添付ファイル（暗号化）
    ├── [SES] ← メール通知
    └── [CloudWatch] ← ログ・監視

[管理者（事務局PC）]
    │
    ▼
[管理画面（同一Next.jsアプリ内、認証分離）]
```

#### 技術スタック

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| Frontend | Next.js 15 + TypeScript + Tailwind CSS | SSR、アクセシビリティ対応、スマホ最適化 |
| Backend | Node.js + Express / Next.js API Routes | 軽量、申請ワークフロー向き |
| DB | PostgreSQL（RDS） | 信頼性、JSON型で柔軟な申請データ保存 |
| ファイル | S3（SSE-S3暗号化） | 添付ファイル管理、10MB対応 |
| 認証 | 自前実装（bcrypt + JWT）or Cognito | パスワード要件対応 |
| CDN/WAF | CloudFront Pro ($15/月) | WAF, DDoS, CDN 一体 |
| メール | Amazon SES | 通知メール配信 |
| 監視 | CloudWatch + GuardDuty | ログ1年保管、不正アクセス検知 |
| CI/CD | GitHub Actions + Docker | 自動デプロイ |

### 6.2 入札価格の検討

R7落札額 **1,245万円** との比較から、コスト構造の最適化が必要。

#### コスト削減シナリオ

| 削減策 | 削減額 | 実現性 |
|--------|--------|--------|
| **BPO体制スリム化**（業務管理者1名 + 業務従事者1名 + パート1名） | ▲400万円/年 | ○ |
| **システム構築の簡素化**（既存OSSテンプレート活用） | ▲150万円 | ○ |
