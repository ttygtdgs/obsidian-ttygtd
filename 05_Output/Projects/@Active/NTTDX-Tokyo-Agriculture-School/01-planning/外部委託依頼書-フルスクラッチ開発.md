# 外部委託依頼書
## 東京都農業高校向け 補助金支給事務Webシステム（フルスクラッチ開発）

**案件名**: 東京都農業高校向け補助金支給事務システム 新規構築
**依頼日**: 2026年2月14日
**回答期限**: 2026年__月__日
**依頼元**: DXPN SD事業部

---

## 1. 依頼内容

東京都の農業高校に通う高校生を対象とした**資格取得補助金の申請・審査・支給を管理するWebシステム**のフルスクラッチ開発を委託したい。

### 委託範囲

| 対象 | 内容 |
|------|------|
| **委託する** | Webアプリケーションの設計・開発・テスト・環境構築・納品 |
| **委託しない** | PM・入札対応（DXPN教育事業部）、BPO業務運用（ネクシア）、AI機能（今回スコープ外） |

### 背景

- R7年度は日本ビジネスクリエイトが1,245万円で受注・運用中（BPO運用込み）
- R8年度以降の入札に向け、システム部分の構築見積を取得したい
- 既存システムの仕様はR7年度をほぼ踏襲する想定

### 座組

```
┌─────────────────────────────────────────────────────┐
│ DXPN 教育事業部                                      │
│  - フロント・入札対応・PM                              │
│  - 東京都との窓口                                     │
├─────────────────────────────────────────────────────┤
│ ネクシア（BPO）           │ 外部パートナー（本依頼先）    │
│  - マニュアル作成         │  - Webシステムの設計・開発   │
│  - 問合せ対応             │  - インフラ構築             │
│  - 申請受付・審査補助     │  - テスト・納品             │
│  - 支払データ作成         │  - 納品後の初期サポート     │
└─────────────────────────────────────────────────────┘
```

## 2. 要望（機能スコープ）

### 2-1. ユーザー側機能（高校生・保護者）

| # | 機能 | 説明 |
|---|------|------|
| U-1 | アカウント登録・ログイン | メールアドレスベースの認証、パスワードリセット |
| U-2 | 補助金交付申請 | 申請フォーム入力、必要書類（学生証等）のアップロード |
| U-3 | 情報変更申請 | 住所・口座等の変更申請、変更書類のアップロード |
| U-4 | 実績報告 | 資格受験後の実績報告、証明書類（合格証・領収書等）のアップロード |
| U-5 | 支払申請 | 額の決定通知後の支払申請、振込口座情報の入力 |
| U-6 | ステータス確認 | 申請状況の一覧表示・進捗確認 |

### 2-2. 管理者側機能（事務局・ネクシア）

| # | 機能 | 説明 |
|---|------|------|
| A-1 | ユーザー管理 | 高校生の一覧表示・検索・詳細表示 |
| A-2 | 申請レビュー | 申請の承認・否認、差し戻しコメント、提出書類の確認 |
| A-3 | 一覧・検索 | 申請のフィルタリング・ソート・全文検索 |
| A-4 | データ出力 | 申請データのCSV/Excel出力、支払データの作成 |
| A-5 | 通知管理 | 申請者へのメール通知（受付・承認・否認・差し戻し） |

### 2-3. 対象外（本依頼に含まない）

- AI機能（チャットボット、OCR、自動整合性チェック等）
- iOS/Androidネイティブアプリ
- 運用・保守の継続契約（別途協議）
- ユーザー向け操作研修

---

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| **可用性** | 稼働率 99.5%以上 |
| **性能** | ページ表示 2秒以内 |
| **セキュリティ** | HTTPS必須、CORS/CSRF対策、個人情報保護法準拠 |
| **認証** | メール+パスワード認証（AWS Cognito等） |
| **対応端末** | PC・スマートフォン・タブレット（レスポンシブ） |
| **ファイル** | PDF・JPEG・PNG のアップロード（1ファイルあたり10MB程度を想定） |
| **データ保持** | 申請データ・書類の5年間保持 |
| **バックアップ** | 日次自動バックアップ |

## 4. 予算

| 項目 | 金額 |
|------|------|
| **システム構築費（税別）** | **560万円前後**（社内概算） |
| 内訳 - 実装工数 | 320h x @10,000円 = 320万円 |
| 内訳 - レビュー工数 | 80h x @10,000円 = 80万円 |
| 内訳 - 利益・バッファ | 160万円（40%） |

- 上記は社内試算の概算であり、妥当性の確認も含めて見積もりを依頼する
- 提示予算を超える場合でも、根拠があれば検討する
- 段階的な開発（フェーズ分割）の提案も歓迎する

### 支払条件（希望）

| タイミング | 割合 | 金額目安 |
|------------|------|----------|
| 着手金 | 30% | 168万円 |
| 中間（設計完了時） | 40% | 224万円 |
| 残金（検収完了時） | 30% | 168万円 |

## 6. 見積回答時にお願いしたい内容

以下の項目を含めて回答をお願いしたい。

1. **見積金額**（税別）と**工数内訳**

2. **想定体制**
   - 人数・役割・スキルセット
3. **開発スケジュール**
   - マイルストーン・納品物の時期
4. **技術スタックの提案**
   - 変更提案がある場合はその理由
5. **納品物一覧**
   - ソースコード、ドキュメント、環境等
6. **納品後サポート**
   - 瑕疵対応期間・範囲
7. **前提条件・リスク**
   - 見積の前提、制約事項、懸念事項
