# 提案1: スコアシートDX化

## 1.1 提案概要

### 課題

```
【現状の業務フロー】

巡回 → 紙スコアシート記入 → PDFスキャン → 手作業でExcel転記 → Dropbox保存
                                               ↑
                                          ここが課題
                                          ・工数負担
                                          ・転記ミス
                                          ・属人化
```

| 課題 | 影響 |
|------|------|
| 転記作業の工数負担 | 担当者の業務時間を圧迫 |
| 転記ミスのリスク | データ品質の低下 |
| 属人化 | 特定担当者への依存、業務継続リスク |

### 解決策

**OCRによる転記作業の半自動化**

```
【提案する業務フロー】

巡回 → 紙スコアシート記入 → PDFスキャン → OCR自動読取 → 確認・修正 → Excel出力
                                               ↑
                                          ここを自動化
                                          ・転記作業80%削減
                                          ・ミス防止
                                          ・標準化
```

### 期待効果

| 指標 | 現状 | 導入後 |
|------|------|--------|
| 転記作業時間 | X時間/月 | **80%削減** |
| 転記ミス率 | 発生あり | **大幅低減** |
| 属人化リスク | 高 | **低減** |

## 2.3 情シス連携の進め方

### なぜ情シス連携が必要か

| 観点 | 情シスとの調整事項 |
|------|-------------------|
| **セキュリティ** | アクセス権限、認証方式、データ分類 |
| **インフラ** | AWS環境へのアクセス権限、Snowflake設定 |
| **運用** | 監視・保守体制、バックアップポリシー |
| **コスト** | インフラ費用の負担、運用費用 |

### 情シス連携の進め方（案）

```
【情シス連携のステップ】

Step 1: 初回顔合わせ
├─ 参加者: 貴社情シス + 現場責任者 + 弊社
├─ 内容: プロジェクト概要説明、技術要件の共有
└─ ゴール: プロジェクトへの理解、協力体制構築

Step 2: 技術要件すり合わせ
├─ 参加者: 貴社情シス + 弊社
├─ 内容: AWS環境確認、Snowflake設定、セキュリティ要件
└─ ゴール: 技術仕様の合意

Step 3: 環境準備
├─ 担当: 貴社情シス（弊社サポート）
├─ 内容: 必要な権限付与、環境セットアップ
└─ ゴール: 開発着手可能な状態

Step 4: 開発・テスト
├─ 担当: 弊社（情シスレビュー）
├─ 内容: インポートスクリプト開発、テスト
└─ ゴール: 本番リリース準備完了
```

### 情シスとの議論ポイント

| # | 議論ポイント | 確認事項 |
|---|-------------|---------|
| 1 | **権限** | 弊社にどこまでのアクセス権限を付与可能か |
| 2 | **データ** | Snowflakeへのデータ格納ポリシー |
| 3 | **セキュリティ** | 外部ベンダーの作業に関するルール |
| 4 | **スケジュール** | 情シス側の対応可能時期 |
