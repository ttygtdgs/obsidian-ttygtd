```

今期                              来期
├─────────────────────────────────┼─────────────────────────────────┤

├─────────┬─────────────────────┤

**判断根拠**

- データ基盤は関係者調整・ゴール設定に時間を要するため、議論を先行させる

# パターンA（OCR先行）詳細骨子

## 1. 提案ストーリーライン

### 1.1 課題認識

| 現状の課題 | 影響 |
|-----------|------|
| 紙スコアシート → Excel転記が手作業 | 工数負担（1件あたり約X分） |
| 転記ミスが発生するリスク | データ品質の低下 |
| 属人化（特定担当者に依存） | 業務継続リスク |

### 1.2 なぜOCRを先行させるか

| 判断ポイント | OCR先行の優位性 |
|-------------|----------------|
| **課題の緊急度** | 日々の転記作業負担を早期に軽減 |
| **効果の測定** | 精度・工数削減率を定量評価可能 |
| **投資判断** | PoC結果でGo/No-Goを明確に判断 |
| **リスク** | PoCで技術検証、リスクを最小化 |

### 1.3 期待効果

| 指標 | 現状 | 目標 |
|------|------|------|
| 転記作業時間 | X時間/月 | **80%削減** |
| 転記ミス率 | X% | **ゼロに近づける** |
| 属人化リスク | 高 | **低減** |

### Week 3-4: Custom Extractor学習

#### 作業内容
| # | 作業 | 詳細 |
|---|------|------|
| 1 | スキーマ設計 | 抽出項目の定義（日付、ID、各測定値、チェック項目等） |
| 2 | アノテーション | 学習データ作成（20-30件を手動ラベリング） |
| 3 | Processor学習 | Custom Extractorの学習実行 |
| 4 | 初期精度評価 | テストデータでの精度測定 |

#### アノテーション基準
| 項目タイプ | ラベリング方法 |
|-----------|---------------|
| テキスト項目 | バウンディングボックス + テキスト値 |
| 数値項目 | バウンディングボックス + 数値 |
| チェックボックス | 位置 + ON/OFF状態 |
| 日付 | バウンディングボックス + 正規化形式 |

#### 精度評価方法
| 評価指標 | 計算方法 | 目標 |
|---------|---------|------|
| 文字認識精度 | 正解文字数 / 総文字数 | 85%以上 |
| 項目抽出精度 | 正解項目数 / 総項目数 | 85%以上 |
| チェックボックス精度 | 正解判定数 / 総チェック数 | 90%以上 |

#### アウトプット
- 学習済みCustom Extractor
- 初期精度レポート
- 改善ポイントの洗い出し

#### システム構成

┌─────────────────────────────────────────────────────────────┐

│                                        ↓                    │

│                               │ ・低信頼項目表示│            │
│                               │ ・修正入力     │            │
│                               │ ・確定操作     │            │
│                               └────────────────┘            │
└─────────────────────────────────────────────────────────────┘
```

# パターンB準備（関係者議論）詳細骨子

## 1. 議論すべきテーマ

### 1.1 データ基盤の目的・ゴール像

| 論点 | 検討内容 |
|------|---------|
| **何のためのデータ基盤か** | 単なる保管？分析？レポート？意思決定支援？ |
| **どこまでのデータを対象とするか** | スコアシートのみ？他データも統合？ |
| **将来的な拡張性** | AI分析、予測、他システム連携の可能性 |

### 1.2 利用者・ユースケース

| 利用者 | 想定ユースケース |
|--------|----------------|
| 現場担当者 | データ検索、履歴確認 |
| 管理者 | 集計レポート、傾向把握 |
| 経営層 | KPIモニタリング、意思決定 |
| （将来）研究部門 | データ分析、パターン発見 |

### 1.3 既存IT環境との整合性

| 確認事項 | 内容 |
|---------|------|
| **既存システム** | AWS、Snowflake、Dropboxの利用状況 |
| **セキュリティポリシー** | データ外部保管可否、アクセス制御要件 |
| **IT運用体制** | 社内IT担当の有無、外部委託状況 |
