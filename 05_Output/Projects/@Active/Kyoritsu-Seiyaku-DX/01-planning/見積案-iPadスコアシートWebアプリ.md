# 見積案：iPadスコアシート入力Webアプリ開発

**プロジェクト名**: 共立製薬様 スコアシートDX化
**提出先**: 社内検討用（NTTDX パートナーズ）
**作成日**: 2026-02-27
**関連仕様**: [[iPad-スコアシート入力アプリ-プロダクト仕様書]]

---

## 1. 見積前提条件

### 1.1 スコープ

| 区分 | 対象 |
|------|------|
| **アプリ形態** | PWA（Progressive Web App） ※iPad Safari対応 |
| **対象フェーズ** | Phase 0（仕様確定）〜 Phase 1（PoC開発）〜 Phase 2（本開発）〜 Phase 3（導入） |
| **対象ユーザー** | 観察者（5〜10名）、試験責任者（1〜2名）、管理者（1名） |
| **動作環境** | iPad（第10世代〜）、Safari / Chrome |

### 1.2 前提・制約

- クライアント側のiPad調達は別途（本見積に含まない）
- 厚労省GCP準拠の監査証跡機能を含む
- オフライン対応（Service Worker + IndexedDB）
- 写真撮影機能はブラウザMediaDevices API利用
- クラウド基盤はGCP（Cloud Run + Cloud SQL + Cloud Storage）
- Snowflake連携は Phase 2 で対応
- プロダクト仕様書の確認事項（27項目）はクライアント確認後に確定

## 4. 見積パターン（松竹梅）

### 4.1 パターン比較

| | 梅（ミニマム） | 竹（推奨） | 松（フルスペック） |
|---|--------------|-----------|------------------|
| **対象フェーズ** | Phase 0 + 1 のみ | Phase 0〜3 全フェーズ | Phase 0〜3 + 追加機能 |
| **主な機能** | スコア入力・保存、写真撮影、オフライン基本対応 | 左記 + 監査証跡、承認フロー、Excel出力、Snowflake連携 | 左記 + ネイティブアプリ化（Swift）、AI異常検知、BIダッシュボード |
| **期間** | 約2ヶ月 | 約7ヶ月 | 約10ヶ月 |
| **費用（税抜）** | **465万円** | **1,675万円** | **2,500〜3,000万円** |
| **メリット** | 低コストで現場検証可能、Go/No-Go判断材料 | GCP準拠で本番運用可能、費用対効果のバランス | 最高のUX、将来拡張性が高い |
| **リスク** | PoCのみで本運用には追加開発必要 | PWAの制約（オフライン安定性等） | 費用大、開発期間長 |

### 4.2 推奨

**竹（推奨）パターン** をベースに、Phase 1（PoC）の結果を踏まえてPhase 2以降のスコープを調整する段階的アプローチを推奨。

```
Phase 0 + 1 で契約  →  PoC結果でGo/No-Go判断  →  Phase 2 + 3 で追加契約
（465万円）                                        （1,210万円）
```

**理由**:
- クライアントのリスクを最小化（PoC段階での投資判断が可能）
- 現場フィードバックを本開発に反映できる
- 稟議が通りやすい（段階的な投資）

## 7. リスクと対応方針
