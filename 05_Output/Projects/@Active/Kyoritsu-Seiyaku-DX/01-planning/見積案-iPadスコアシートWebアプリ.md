## 1. 見積前提条件

### 1.1 スコープ

iPadからスコアシート（ワクチン効力試験の観察記録13項目）を入力し、写真を添付して、AWSのRDS（PostgreSQL）に保存するWebアプリケーション。全操作の監査証跡を自動記録する。

| 区分 | 対象 |
|------|------|
| **アプリ形態** | Webアプリ（iPad Safari / Chrome 対応） |
| **主要機能** | スコア入力 + 写真撮影 + 監査証跡 + AWS RDS保存 |
| **対象ユーザー** | 観察者（5〜10名） |
| **動作環境** | iPad（第10世代〜）、Wi-Fi接続環境 |
| **クラウド基盤** | AWS（ECS or Lambda + RDS PostgreSQL + S3） |

### 1.2 スコープ内

- スコアシート入力画面（13項目 + 体温 + 体重 + 備考）
- 入力データのAWS RDS（PostgreSQL）への保存
- 試験・個体の選択画面
- 基本的な入力バリデーション（範囲チェック等）
- iPad向けUIの最適化
- **写真撮影・添付機能**（iPadカメラ → S3保存）
- **監査証跡（Audit Trail）**（全操作の記録、修正時の理由入力必須）

### 1.3 スコープ外（本見積に含まない）

- ユーザー認証・ログイン機能
- オフライン対応
- 承認ワークフロー
- Excel / PDF出力
- Snowflake連携
- iPad端末の調達

### 1.4 人月単価の前提

| ロール | 単価（税抜） |
|--------|-------------|
| PM / PL | 150万円/月 |
| エンジニア | 100万円/月 |

## 6. 監査証跡の設計方針

GCP（厚労省GCP）準拠の監査証跡として、以下を記録する。

| 記録項目 | 内容 |
|---------|------|
| 操作種別 | INSERT / UPDATE / DELETE |
| 操作者 | 端末識別 or ユーザー名（認証導入後） |
| 操作日時 | タイムスタンプ（UTC + JST） |
| 変更項目 | 変更されたフィールド名 |
| 変更前の値 | 修正前のデータ |
| 変更後の値 | 修正後のデータ |
| 変更理由 | **修正時は理由の入力を必須** |

- 監査証跡テーブルは追記のみ（UPDATE / DELETE不可）
- アプリケーションレベルでの改ざん防止
