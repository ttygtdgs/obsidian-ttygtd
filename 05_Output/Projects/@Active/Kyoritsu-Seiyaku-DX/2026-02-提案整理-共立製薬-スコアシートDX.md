# 提案1: スコアシートDX化（スマートデバイス入力）

## 1.1 課題認識

### 現状の業務フロー

```
巡回 → 紙スコアシート記入 → PDFスキャン → 手作業でExcel転記 → Dropbox保存
                                               ↑
                                          ここが課題
                                          ・工数負担
                                          ・転記ミス
                                          ・属人化
```

| 課題 | 影響 |
|------|------|
| 転記作業の工数負担 | 担当者の業務時間を圧迫 |
| 転記ミスのリスク | データ品質の低下、ローデータとしての信頼性に影響 |
| 属人化 | 特定担当者への依存、業務継続リスク |
| 紙の保管管理 | 物理的な保管スペース、紛失・劣化リスク |

---

## 1.2 方向転換：OCR → スマートデバイス入力

### 検討経緯

当初はOCR（紙→スキャン→自動読取）による転記自動化を検討していたが、先方との議論を経て**スマートデバイスでの直接入力**に方向転換。

| 観点 | OCR方式 | スマートデバイス入力 |
|------|---------|---------------------|
| **紙の工程** | 残る（紙→スキャン→OCR） | **なくなる** |
| **転記作業** | 自動化（ただし確認作業は残る） | **そもそも不要** |
| **データ精度** | OCR精度に依存（85%目標） | **入力時にバリデーション可能** |
| **リアルタイム性** | バッチ処理 | **即時反映** |
| **ローデータ保管** | 紙が原本、電子は複製 | **電子データが原本になる** |
| **導入ハードル** | 低（既存フロー維持） | スマートデバイスの現場配備が必要 |

**方向転換の理由**
- 先方に紙へのこだわりがない
- ローデータの保管が最重要 → 電子データを原本とする方が管理しやすい
- OCR方式では紙→スキャン→OCR→確認という工程が残り、根本解決にならない
- スマートデバイス入力なら、入力～保管まで一気通貫でデジタル化できる

---

## 1.3 新しい業務フロー

```
【提案する業務フロー】

巡回 → スマートデバイスで直接入力 → データ保存（クラウド） → Excel出力 / ダッシュボード
              ↑                              ↑
         ここをデジタル化                ローデータとして
         ・入力フォーム                  電子保管
         ・バリデーション               （GCP準拠）
         ・オフライン対応
```

### 期待効果

| 指標 | 現状 | 導入後 |
|------|------|--------|
| 転記作業 | 手作業でExcel転記 | **完全不要** |
| データ精度 | 転記ミスあり | **入力時バリデーションで担保** |
| データ反映 | バッチ（転記後） | **リアルタイム** |
| ローデータ保管 | 紙保管 | **電子保管（監査証跡付き）** |
| 属人化リスク | 高 | **標準化された入力フォーム** |

---

## 1.4 ローデータ保管要件（厚労省GCP準拠）

先方がこだわるローデータの保管について、厚労省のGCPガイドライン（医薬品の臨床試験の実施の基準）に基づき、電子データを原本とする場合の要件を整理。

> 参考: https://www.mhlw.go.jp/web/t_doc?dataId=00tc5323&dataType=1&pageNo=

### 電子データを原資料（ローデータ）とするための要件

| # | 要件 | 対応方針 |
|---|------|---------|
| 1 | **システム完全性の保証** | クラウド基盤（AWS/GCP）による可用性・信頼性の確保 |
| 2 | **データ修正履歴の記録** | 全入力・修正操作の監査証跡（Audit Trail）を自動記録 |
| 3 | **セキュリティシステム** | ユーザー認証、権限管理、アクセスログ |
| 4 | **バックアップ体制** | 自動バックアップ、災害復旧計画 |
| 5 | **データ修正者名簿** | 操作者の特定・管理（ユーザーアカウント管理） |

### スマートデバイス入力のメリット（ローデータ観点）

- **監査証跡が自動記録**: 誰が・いつ・何を入力/修正したか全て記録
- **改ざん防止**: 修正履歴が残るため、紙よりもデータの信頼性が高い
- **保管の確実性**: クラウド保管により紛失・劣化リスクを排除
- **検索性**: 必要なデータを即座に検索・抽出可能

# 提案2: データ基盤構築

## 2.1 提案概要

### 目的

Dropbox上に散在するExcelデータを、Snowflakeに構造化データとして格納し、検索・分析可能なデータ基盤を構築する。

```
【現状】                              【目指す姿】

Dropbox上にExcelが散在               データ基盤で一元管理
                                             ↓
┌─────────────────┐                 ┌─────────────────────────┐
│ folder_A/       │                 │                         │
│  ├─ 2024-01.xlsx│                 │    Snowflake            │
│  ├─ 2024-02.xlsx│    ────→       │    (データ基盤)          │
│ folder_B/       │                 │                         │
│  ├─ 2024-03.xlsx│                 │    ・構造化データ       │
│  └─ ...         │                 │    ・検索・分析可能     │
└─────────────────┘                 │    ・アクセス管理       │
                                    └─────────────────────────┘
・データ散逸                                    ↓
・検索困難                           ┌─────────────────────────┐
・活用困難                           │   BIダッシュボード      │
                                    │   データ検索UI          │
                                    └─────────────────────────┘
```

### 期待効果

| 観点 | 効果 |
|------|------|
| **データ管理** | 散逸防止、一元管理、バージョン管理 |
| **データ活用** | 検索・フィルタリング、集計・分析、傾向把握 |
| **業務効率** | データ探しの時間削減、レポート作成効率化 |
| **将来拡張** | AI分析、予測、他システム連携の基盤 |

## 2.4 提案①②の連携

提案①（スマートデバイス入力）と提案②（データ基盤）は、将来的に連携可能。

```
【連携イメージ】

提案①                                   提案②
┌───────────────────┐                   ┌───────────────────┐
│ スマートデバイス   │                   │                   │
│ 入力               │ ──── 直接投入 ──→│  Snowflake        │
│                   │                   │  (データ基盤)     │
│ ・スコアシート入力 │                   │                   │
│ ・ローデータ保管   │                   │  ・一元管理       │
│                   │                   │  ・分析・検索     │
└───────────────────┘                   └───────────────────┘
```

| 連携メリット | 内容 |
|-------------|------|
| **転記工程の完全排除** | デバイス入力 → データ基盤に直接格納（Excelを経由しない） |
| **リアルタイム分析** | 入力と同時にダッシュボードに反映 |
| **ローデータの一元管理** | 入力データ＝ローデータとしてSnowflakeで保管 |
