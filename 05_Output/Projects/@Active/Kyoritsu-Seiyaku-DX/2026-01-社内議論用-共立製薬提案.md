## 3. 相談①：スコアシートDX化（OCR）

### 現状の運用フロー

```
巡回して紙のスコアシートを記入 → Excelに転記（手作業）
```

### 本番運用フロー（想定）

```
紙のスコアシート → スキャン（PDF化）→ OCR処理 → 確認UI → Excel出力
```

### PoC / 本番のスコープ

| 機能 | PoC | 本番 | 備考 |
|------|:---:|:---:|------|
| OCR精度評価 | ○ | - | サンプルPDF（10-20件）で検証 |
| 精度レポート・Go/No-Go判断 | ○ | - | |
| PDFアップロード機能 | - | ○ | |
| OCR処理（Document AI） | ○ | ○ | |
| 確認UI（低信頼度ハイライト・手動補正） | ○ | ○ | PoCでも簡易版を実装 |
| Excel出力（既存フォーマット準拠） | - | ○ | |

### 【相談】スコープの切り方

上記の表の通り、PoCで確認UIまで含める形で進めたいが、問題ないか？

### 【相談】環境について

- クライアントはAWSを利用中
- ただし、OCR精度が高いと思われる **GCP（Document AI）** で検証を進めたい
- PoCの評価が良ければ、弊社GCP環境で納品しランニング費用をいただく形でよいか？

### 【確認】OCRエンジンについて

- OCR精度が高いのはGoogle（Document AI）という認識で合っているか？
  - 特に手書き認識において優位と想定

### 【相談】精度の合格基準

- PoCで「精度X%以上なら本開発に進む」という基準を事前に定めておくか？
- 定める場合、どのレベルが妥当か？（例：項目抽出精度 85%以上）

## 4. 相談②：データ基盤構築

### 先方の環境

- AWS・Snowflakeを利用中

### 本番運用フロー（想定）

```
Excel（スコアシート）→ 格納スクリプト → Snowflake → 自然言語検索UI
```

※PoC/本番の詳細スコープは今後検討（今回は方向性の相談）

### 実装内容

- ExcelデータをSnowflakeへ格納するスクリプトを構築
- 自然言語でデータ検索できるUI（チャットUI等）を提供
  - ITリテラシーが低めの部署でも使える形でデータを引き出せるようにする

### 【相談】

- この方向性で問題ないか？
- 費用・ビジネスモデルについて
  - 先方の社内環境に構築する形になると想定
  - 権限をもらい、先方内の環境で構築
  - 情シスとの議論部分は手弁当（無償）、構築部分から費用を取る形でよいか？
- 社内でSnowflakeなどデータ基盤構築の経験・実績はあるか？
  - 知見があれば共有してほしい
