# 🧭 都立農業系高校生資格取得支援制度 補助金申請システム  
## 現状フローと課題整理（申請者側／管理者側）

---

## 🧩 現状の業務フロー（概要）

本制度の申請は、**申請者側（生徒・保護者）** と **管理（審査）側（事務局）** に分かれ、  
オンライン上の「マイページ」を介して進行する。

---

### 🧑‍🎓 申請者側フロー

1. **マイページ登録**
   - エントリーフォームから個人情報を入力。
   - 登録完了メールでマイページURLを受領。

2. **補助金交付申請**
   - マイページへログイン。
   - 資格名・受験日・学校情報などを入力。
   - 学生証画像をアップロード。
   - 保護者口座の場合は代理人情報を追加。
   - 内容確認 → 送信。
   - 受付メールを受信。

3. **変更承認申請（必要時）**
   - 受験資格・日程変更がある場合、変更フォームに入力。
   - 内容確認 → 送信。
   - 承認結果メールを受信。

4. **実績報告・証明書提出**
   - 受験完了後、受験料・教材費などを入力。
   - 領収書・レシートをアップロード。
   - 送信 → 額の確定通知を受領。

5. **支払申請**
   - 「額の決定通知書」受領後、支払フォームで口座情報を入力。
   - 通帳画像をアップロード。
   - 内容確認 → 送信。
   - 支払完了メールを受信 → 振込処理。

---

### 🧑‍💼 管理（審査）側フロー

1. **申請受付・内容確認**
   - 新規申請データを受信。
   - 内容チェック（資格・学校・書類）を実施。
   - 不備があれば修正依頼、問題なければ次工程へ。

2. **交付審査・決定**
   - 補助対象の適格性を審査。
   - 承認後、交付決定通知書を送信。

3. **変更申請対応**
   - 変更承認申請を審査。
   - 妥当であれば承認通知書を発行。

4. **実績報告審査**
   - 提出書類（領収書・経費明細）を確認。
   - 不備があれば再提出依頼、問題なければ額の決定通知書を発行。

5. **支払処理**
   - 支払申請内容を確認。
   - 金額・口座情報を照合。
   - 振込手続き実施 → 支払完了通知送信。

---

### 🔁 双方の主な連携ポイント

| フェーズ | データ／通知のやり取り | 主な媒介 |
|-----------|-------------------------|------------|
| 登録 | 登録完了・申請URL送信 | 自動メール |
| 申請 | 受付・決定通知 | 自動メール |
| 変更 | 承認結果通知 | 自動メール |
| 実績報告 | 額の決定通知 | 自動メール |
| 支払 | 支払完了連絡 | 振込＋メール |

## 💭 不満・課題整理

---

### 🧑‍🎓 申請者側の不満・課題

| カテゴリ | 不満・課題内容 | 背景・影響 |
|-----------|----------------|-------------|
| 操作の分かりにくさ | - マニュアルが長く複雑で理解に時間がかかる。<br>- どのフォームを使えばよいか直感的に分からない。 | ページ構造が多層的（マイページ／申請フォーム／変更フォームなど）。 |
| 入力作業の煩雑さ | - 同じ情報を複数回入力する必要がある。<br>- 入力条件（代理人・住所など）が分かりづらい。 | 入力データの再利用・自動入力機能が未実装。 |
| ファイルアップロードの不便さ | - 10MB制限などのエラーが発生しやすい。<br>- スマホ操作でリサイズが難しい。 | ファイル制限の説明がUI上で不十分。 |
| ステータスの不透明さ | - 申請後の進捗（審査中／承認済み）が分からない。<br>- メール通知のみで履歴が見づらい。 | マイページ上での進捗可視化がない。 |
| 問い合わせ対応の不便さ | - 電話・メールのみで時間がかかる。<br>- 担当ごとの連絡が煩雑。 | チャットやAIナビゲーション機能が未導入。 |
| 書類確認の不安 | - 提出した書類が有効か判断できない。<br>- 再提出指示があっても修正点が不明確。 | 審査基準が申請者に共有されていない。 |

---

### 🧑‍💼 管理（審査）側の不満・課題

| カテゴリ | 不満・課題内容 | 背景・影響 |
|-----------|----------------|-------------|
| 書類確認の手間 | - 画像確認（学生証・領収書など）が膨大。<br>- 不鮮明な画像対応に時間がかかる。 | OCRや自動認識機能がない。 |
| 不備対応の非効率 | - 不備対応をメールで個別処理。<br>- 再提出管理が煩雑。 | 不備検出・再提出フローが自動化されていない。 |
| データ集計の煩雑さ | - 学校別／資格別などを手作業で集計。 | システムにBI・分析機能がない。 |
| 進捗管理の難しさ | - 案件全体の進行状況を一目で把握できない。 | ステータス管理機能が限定的。 |
| 問い合わせ対応の負担 | - 進捗確認・修正依頼の問い合わせが多い。 | 自動応答・FAQ機能が未整備。 |
| 重複・誤登録の検出困難 | - 同一生徒による二重申請を手動確認。 | 照合機能・重複検出が未実装。 |
| 通知文面・証明書作成の負担 | - 通知書・メール文を手作成。 | 自動生成機能が未導入。 |

---

### 🔁 共通の課題（申請者・管理者双方）

| 観点 | 不満内容 | 背景 |
|------|-----------|------|
| システムUIの複雑さ | 操作手順・画面遷移が分かりづらい。 | ページ構造と機能が分散。 |
| 進捗ステータスの追跡性 | 双方で申請進捗をリアルタイムに把握できない。 | ステータス同期・自動通知が未実装。 |
| コミュニケーションの断絶 | メール中心でやりとりが煩雑。 | 問い合わせ履歴の共有・一元管理なし。 |
| マニュアル依存 | 操作がドキュメント前提で分かりにくい。 | 動的ヘルプ・ガイド機能が未導入。 |
