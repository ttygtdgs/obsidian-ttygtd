# 札幌市アイデアソン - データセット統合整理

## 📋 概要

札幌市アイデアソンで利用するデータセットの各データの中身を統合整理したドキュメント。

### 使用データセット

1. **データスタジアム提供データ**（スポーツデータ3種）
   - プロ野球
   - Jリーグサッカー
   - Bリーグバスケットボール

2. **iタウンページデータ**（事業所・商圏データ2種）
   - 標準出力データ（個票データ）
   - 統計情報データ（集計データ）

**データ提供元**: さっぽろ市データ圏  
**データ期間**: 未確認（要確認）  
**データ形式**: CSV/JSON/固定長（推測、要確認）  
**データサイズ**: 未確認（要確認）

## 📊 プロ野球データ

### データセット構成

1. **試合データ**（30項目）
2. **選手成績データ（打者）**（44項目）
3. **選手成績データ（投手）**（詳細項目、約150項目以上）
4. **イベントデータ**（約200項目）
5. **選手プロフィールデータ**（25項目）

### 1. 試合データ（30項目）

#### 基本情報
- `game_id`: 試合ID
- `year_id`: 年度ID
- `game_kind_id/name`: 試合種別（レギュラーシーズン/ポストシーズンなど）
- `game_date`: 試合日付
- `game_year/month`: 試合年度・月

#### スタジアム・チーム情報
- `stadium_id/name`: スタジアムID・名称
- `home_team_*`: ホームチーム情報（リーグID、リーグ名、チームID、チーム名）
- `away_team_*`: アウェイチーム情報（リーグID、リーグ名、チームID、チーム名）

#### 試合結果・時間情報
- `game_started_time/ended_time`: 試合開始・終了時刻
- `game_time`: 試合時間
- `game_delayed_time`: 試合中断時間
- `attendance`: **観衆（観客動員数）** ← マーケティング施策の効果測定に重要
- `home_team_score/away_team_score`: ホーム・アウェイチーム得点
- `inning`: イニング（試合終了時）
- `top_bottom_id/name`: 表裏ID・名称（試合終了時）
- `game_end_status`: 試合終了状況

#### コンテンツ情報
- `preview`: **見どころ** ← マーケティングコンテンツへの活用
- `review`: **戦評** ← コンテンツ生成への活用

### 2. 選手成績データ（打者）- 44項目

#### 打撃成績
- `hr`: **本塁打** ← インパクトのある可視化
- `sb`: **盗塁** ← スピード・積極性の可視化
- `h`: 安打
- `rbi`: 打点
- `bb`: 四球
- `so`: 三振
- その他多数の打撃統計

### 3. 選手成績データ（投手）- 詳細項目

#### 主要項目
- 試合基本情報（game_id, year_id, game_date等）
- チーム情報（team_id, team_name等）
- 選手情報（player_id, player_name等）
- 投球成績（投球数、被安打、被本塁打、奪三振、四球等）
- 詳細な投球データ（約150項目以上）

### 4. イベントデータ - 約200項目

#### 4.1 投球位置および投球ゾーン
- `pitch_zone`: 1〜25の値（ストライクゾーン内）
- `pitch_location_x`: -151.2〜151.2（横方向の位置）
- `pitch_location_y`: -31.5〜346.5（縦方向の位置）
- `pitch_target_location_x/y`: 目標位置

**仕様**: センターからホームプレートに向いた視点で記録

#### 4.2 打球位置
- `batted_ball_location_x`: -290〜290（横方向）
- `batted_ball_location_y`: -50〜400（縦方向）
- `batted_ball_grounded_location_x/y`: グラウンド上の位置

#### 4.3 打球方向
- `batted_ball_direction`: 角度（センター方向を0、ライト方向を正、レフト方向を負）
- `batted_ball_direction_3`: 3方向分類（1〜3）
- `batted_ball_direction_5`: 5方向分類（1〜5）

#### 4.4 イベントデータの詳細項目
- 投球フラグ（is_pitch, is_ball, is_strike等）
- 打球フラグ（is_gb, is_fb, is_ld等：ゴロ/フライ/ライナー）
- 打球の強さ（is_soft, is_med, is_hard：C/B/Aランク）
- 打球方向（is_pull, is_straight, is_oppo：引っ張り/センター/逆方向）
- その他多数の詳細フラグ

### 5. 選手プロフィールデータ

- `season_year`: シーズン年度
- `team_league_id`: チームリーグID
- `team_league_name`: チームリーグ名（セ・パ）
- `team_id`: チームID
- `team_name`: チーム名
- `player_id`: 選手ID
- `player_name`: **選手名** ← 選手の特徴分析、コンテンツ生成
- `uniform_number`: 背番号
- `position_type`: ポジションタイプ
- `throws`: 投球（右投げ/左投げ）
- `bats`: 打席（右打ち/左打ち/両打ち）
- `height`: **身長** ← 身体能力の可視化
- `weight`: **体重** ← 身体能力の可視化
- `birthday_date`: 生年月日
- `high_school_id`: 高校ID
- `high_school_name`: **高校名** ← 出身高校の可視化、地域連携
- `college_id`: 大学ID
- `college_name`: **大学名** ← 出身大学の可視化、地域連携
- `company`: 会社（社会人野球経歴）
- `draft_year`: ドラフト年
- `draft_round`: ドラフト順位
- `born_country`: 出身国
- `born_local`: **出身地** ← 出身地の可視化、地域連携
- `blood_type`: 血液型
- `salary`: **年俸** ← 年俸の可視化、選手価値の可視化

### 活用アイデア例
- 観客動員数と対戦カード・時間帯・スタジアムの関係性分析
- 見どころデータを活用したSNS投稿自動生成
- 選手の特徴を数値で見える化（本塁打数、盗塁数など）
- 投球位置・打球位置の可視化コンテンツ
- インパクトのある映像コンテンツ生成
- 選手のプロフィールを可視化（身体能力、出身地、出身校）
- 出身地・出身校の可視化（地域連携マーケティング）
- 年俸と成績の相関分析

## ⚽ Jリーグサッカーデータ

### データセット構成

1. **トラッキングデータ**（秒間25フレームの位置情報）
2. **イベントデータ**（約200項目）
3. **成績データ**（選手・チーム、試合毎・シーズン毎）
4. **試合情報データ**
5. **選手プロフィールデータ**

### 1. トラッキングデータ

#### 基本情報
- `GameID`: 試合を一意に特定するID
- `Frame`: トラッキングシステムのフレーム番号（秒間25フレーム）
- `HA`: ホーム/アウェイ特定用のフラグ（1=ホーム、2=アウェイ、0=ボール）
- `No`: 選手の背番号（ボールの場合は0）

#### 位置・速度情報
- `X`: **ピッチ上のX座標**（-5250～+5250、105m側） ← 選手・ボールの位置可視化
- `Y`: **ピッチ上のY座標**（-3400～+3400、68m側） ← 選手・ボールの位置可視化
- `Speed`: **移動速度**（km/h、例：29.3=29.3km/h） ← 選手のスピード可視化、インパクトのあるコンテンツ

**座標の仕様**:
- ピッチサイズ: 105m × 68m
- X座標: -5250～+5250（105m側）
- Y座標: -3400～+3400（68m側）
- ボールのX/Y: イベントデータから計算した推定位置

### 2. イベントデータ - 約200項目

#### 2.1 基本情報（約20項目）
- `試合ID`: 試合を一意に特定するID（試合日YYYYMMDD+2桁の8桁）
- `履歴No`: 該当試合の全イベントを時系列にした履歴のNo
- `試合状態ID`: 現在の試合の状態ID（1=前半、2=後半）
- `チームID/チーム名`: 該当チームのID・名称
- `選手ID/選手名/選手背番号`: 該当プレーの選手情報
- `ポジションID`: 1=GK、2=DF、3=MF、4=FW、0=不明
- `アクションID/アクション名`: 該当選手のアクション情報
- `ボールＸ/ボールＹ`: **該当イベントのボール位置座標** ← ボール位置の可視化
- `攻撃方向`: 該当チームが攻撃している方向
- `ハーフ開始相対時間`: 前半or後半開始からの時間（hh:mm:ss）

#### 2.2 プレーフラグ（約50項目）

**得点・攻撃関連**
- `F_ゴール`: **ゴールが決まったシュートまたはオウンゴール** ← 得点シーンの可視化
- `F_アシスト`: **ゴールが決まったシュートの前のパスなどが意図的だった場合に1** ← アシストの可視化
- `F_シュート`: **シュートの場合に1** ← シュートシーンの可視化
- `F_パス`: 味方選手にボールをつなぐことを目的としたプレー
- `F_クロス`: F_パス=1かつクロスに該当する場合に1
- `F_スルーパス`: F_パス=1かつスルーパスに該当する場合に1
- `F_ドリブル`: **トラップは含まない、プレーをドリブルと判断した場合** ← ドリブルの可視化、選手の特徴分析
- `F_トラップ`: ドリブルは含まない
- `F_ペナルティエリア進入`: 次の履歴が敵陣ペナルティエリア内で自らor味方選手がプレー出来た場合に1
- `F_30mライン進入`: 次の履歴が敵陣のゴールラインから30m内の場合に1

**守備関連**
- `F_インターセプト`: 相手のパスに対して能動的に動いてそのパスをカット
- `F_ブロック`: 比較的近距離で相手チームのシュートやパスなどを身体に当てて受動的に防いだプレー
- `F_キャッチ`: キーパーのすべてのキャッチ
- `F_ファウル/F_被ファウル`: ファウル（をする/を受ける）
- `F_イエローカード/F_レッドカード`: カード（を受ける）

**攻撃切り替え関連**
- `F_ボールロスト`: インプレーにおいて自チームの攻撃から相手チームの攻撃に切り替わった最後のプレー
- `F_ボールゲイン`: インプレーにおいて相手チームの攻撃から自チームの攻撃に切り替わった最初のプレー

#### 2.3 前後プレー情報（約30項目）
- `B1_*`: 1つ前のイベント情報（履歴No、チームID、選手ID、アクションID、ボール位置等）
- `A1_*`: 1つ後のイベント情報
- `B2_*, A2_*`: 2つ前/後のイベント情報

#### 2.4 その他の詳細項目（約100項目）
- **攻撃情報**: 攻撃履歴No、攻撃開始/終了履歴No、攻撃経過時間、攻撃経過プレー数
- **位置情報**: 縦位置（1～6）、横位置（1～4）、位置座標X/Y、ペナルティエリアF
- **方向情報**: 方向4（前・後・右・左）、方向角度
- **時間情報**: 時間帯（5分単位）、絶対時間、絶対時間ミリ秒
- **プレー詳細**: F_成功、距離、タッチ種別、フレーム番号
- **その他**: Insゴール、Insシュート、点差、相手選手ID、部位ID、フィードIDなど

**主要アクションID例**:
- 15: シュート
- 16: GK
- 17: 直接FK
- 18: キャッチ
- 20: PK
- 21: CK（コーナーキック）
- 22: 間接FK
- 26: キックオフ
- 35: ドリブル
- 36: スルーパス
- 41: インターセプト
- 42: クリア
- 43: ブロック
- 44: スローイン
- 45: クロス
- 50: トラップ
- 74: タックル
- 75: フリックオン

### 3. 成績データ（選手・チーム）

#### 3.1 シュート関連（約20項目）
- `シュート`: 得点を取ることを目的としたプレー
- `ゴール`: **シュートに対するゴール数（オウンゴール除く）** ← ゴール数の可視化
- `成功率％`: ゴール÷シュート
- `枠内/枠内率％`: シュートが枠内に飛んだもの
- `PA内シュート/PA外シュート`: ペナルティエリア内/外からのシュート
- `PKシュート/FKシュート/CKシュート`: ペナルティキック/フリーキック/コーナーキックでの直接シュート
- `ワンタッチシュート/ワンタッチゴール`: ワンタッチでのシュート/ゴール

#### 3.2 パス関連（約30項目）
- `パス/パス成功`: 味方選手につなげる意図があるボールのリリース
- `パス成功率％`: **パス成功÷パス** ← パス精度の可視化
- `ゴール数＋アシスト数`: **ゴール数＋アシスト数（オウンゴール除く）** ← 攻撃貢献度の可視化
- `ワンタッチパス`: ワンタッチでのパス
- `ショートパス/ミディアムパス/ロングパス`: 到達位置までの距離が15m未満/15m以上30m未満/30m以上のパス
- `前パス/右パス/左パス/後パス`: チームの攻撃方向に対して前方向/右方向/左方向/後方向90度のパス
- `クロス`: **相手陣ペナルティエリア内の味方選手にシュートを打たせる狙いがあり、サイドから送られたパス** ← クロスの可視化
- `スルーパス`: **味方が相手最終ラインの裏に走り込むスペースを狙ったパス** ← スルーパスの可視化

#### 3.3 ドリブル・キープ関連（約10項目）
- `ドリブル`: **守備側選手と対峙し、その選手を抜こうとするプレー** ← ドリブルの可視化、選手の特徴分析
- `ドリブル成功`: ドリブルの仕掛け後にパスやシュートなどのプレーが出来た場合
- `ドリブル次シュート/ドリブル次クロス`: ドリブル後に自らシュート/クロス
- `ドリブル保持時間合計`: ドリブルから次のプレーまでの合計時間（秒数）
- `トラップ`: ドリブル以外のボールキープ
- `キャリー`: トラップ地点から次のプレー地点までの直線距離で20m以上移動したキープ

#### 3.4 守備関連（約20項目）
- `セーブ`: **ゴールキーパーが枠内に飛んできたシュートを防いだプレー** ← GKのセーブ可視化
- `セーブ率％`: セーブ数÷（セーブ数＋失点）
- `完封試合数/クリーンシート`: 無失点を記録した試合数
- `キャッチ`: キャッチの前のプレーの内容や、それが敵味方どちらのプレーかは問わない
- `ハンドクリア`: キーパーが手でボールを（キャッチせず）弾いたプレー
- `クリア`: 外に蹴り出し陣地を回復させるなど味方につなぐ意図がなく危険な状態の回避を目的として行ったプレー
- `ブロック`: 自分でボールをコントロールできない状態で、相手のシュートやパスなどを身体に当てて受動的に防いだプレー
- `インターセプト`: **相手のパスに対して能動的に動いてそのパスをカットし、自ら保持もしくは味方につなげたプレー** ← インターセプトの可視化
- `タックル/タックル奪取`: 相手選手がコントロールしているボールを、身体あるいはボールへの接触によって足下から離すプレー

#### 3.5 その他の成績項目
- **進入関連**: PA進入、PA脇進入、30mライン進入
- **ポゼッション関連**: ボール保持時間、ボール保持率％、プレー
- **ゲイン・ロスト関連**: ゲイン、ロスト、リゲイン
- **チームスタイル関連**: ショートカウンター、ロングカウンター、相手陣ポゼッション、自陣ポゼッション
- **結果関連**: 先制試合数、先制勝ち、被先制試合数、逆転勝ち、逆転負け
- **先発ポジション関連**: GK、CB、SB、WB、CH、SH、OH、CFなど

### 4. 試合情報データ

- `試合種別ID`: 2=J1、6=J2、68=J3
- `ホームPK戦得点`: PK戦が行われた場合のホームチームのPK成功数
- `天候１ID/天候２ID/天候３ID`: 0=不明、1=晴れ、2=曇り、3=雨、4=雪、5=雷雨、9=霧、10=屋内

### 5. 選手プロフィールデータ

- `シーズンID`: 該当のシーズンのID
- `チームID`: 該当選手所属チームのチームID
- `選手ID`: 該当選手のID
- `背番号`: 該当選手の背番号
- `選手名`: **該当選手の選手名** ← 選手の特徴分析、コンテンツ生成
- `ポジション`: 該当選手のポジション（GK、DF、MF、FWなど）
- `国籍`: 該当選手の国籍
- `生年月日`: 該当選手の生年月日
- `身長`: **該当選手の身長（cm）** ← 身体能力の可視化
- `体重`: **該当選手の体重（kg）** ← 身体能力の可視化
- `在籍フラグ`: 該当選手がチームに所属しているかのフラグ（0=退団・移籍済み、1=在籍）

### 活用アイデア例
- 選手の動きをリアルタイムで可視化した映像コンテンツ
- 選手のスピードをインパクトのあるビジュアルで表現
- プレーの流れを可視化した映像
- ゴールシーンの可視化
- 選手の特徴を数値で見える化（ドリブル数、パス成功率、ゴール数など）
- 観客動員データから人気カードを分析
- 天候別の観客動員分析

## 🏀 Bリーグバスケットボールデータ

### データセット構成

1. **イベントデータ（座標付き）**

### 1. イベントデータ（座標付き）

#### 1.1 基本情報（約20項目）
- `試合ID`: 該当試合ID
- `履歴No`: 該当試合のプレイを時系列にした履歴ナンバー
- `ピリオド`: 該当プレイのピリオド（クォーター）
- `ピリオド残時間`: 該当プレイのピリオド残り時間
- `チームID/チーム名`: 該当プレイ選手のチーム情報
- `選手名1/選手名2`: **該当プレイの選手名（一人目/二人目）** ← 選手の特徴分析、コンテンツ生成
- `背番号1/背番号2`: 該当選手名1/2の背番号
- `選手ID1/選手ID2`: 該当プレイの選手名1/2のID
- `プレイテキスト`: **該当プレイの内容** ← プレー内容の可視化
- `スコア`: 該当プレイによって得点した場合の両チームのトータルスコア
- `ホームアウェイ`: 該当プレイのホーム/アウェイ（1=ホーム、2=アウェイ）
- `ピリオド終了フラグ/試合終了フラグ`: ピリオド終了時/試合終了時に「1」

#### 1.2 アクション情報（3項目）
- `アクション1`: 該当プレイのアクションID（主要アクション）
- `アクション2`: 該当プレイのアクションID（補助アクション1）
- `アクション3`: 該当プレイのアクションID（補助アクション2）

**主要アクションID例**:
- 1: 3Pシュート○
- 2: 3Pシュート×
- 3-6: 2Pシュート（アウトサイド/インサイドペイント、○/×）
- 7-8: フリースロー（○/×）
- 9: ディフェンスリバウンド
- 10: オフェンスリバウンド
- 11: ブロックショット
- 12: アシスト
- 13: ターンオーバー
- 14: スティール
- 29: **ダンク** ← インパクトのあるコンテンツ
- 35: ファストブレイク（速攻）

#### 1.3 座標情報（座標付きイベントデータのみ）
- `エリア`: **該当アクションが発生したエリア（図1のエリア番号に紐付く）** ← プレー位置の可視化
- `X座標`: **該当アクションが発生したx座標（0～100、コート左下が(0,0)、右上が(100,100)）** ← プレー位置の可視化
- `Y座標`: **該当アクションが発生したy座標（0～100）** ← プレー位置の可視化
- `サイド`: 該当アクションが発生した際のオフェンス側の攻撃方向（left/right、前半と後半で入れ替わる）

**座標の仕様**:
- コートの左下を(x,y):(0,0)、右上を(x,y):(100,100)とする
- B.LEAGUEのコートの長さは28m、エンドラインから3PTラインのトップまでの長さは8.325m
- 例: 左コートの3PTラインのトップのx座標 = 8.325 / 28 * 100 = 29.7
- **注意**: 「エリア、x座標、y座標、サイド」は「アクション」がシュートの時のみデータが入る

### 2. イベントデータ（座標なし）

座標情報を含まないイベントデータ。基本情報とアクション情報のみ。

### 3. 選手成績データ（試合毎）

#### 3.1 基本情報（約10項目）
- `試合ID`: 該当試合ID
- `ピリオド区分`: ピリオド(クォーター)区分（1～4=各ピリオド、5=延長戦、6=再延長戦、18=試合全体）
- `ホームアウェイ`: 該当選手のホーム/アウェイ（1=ホーム、2=アウェイ）
- `チームID/チーム名`: 該当選手のチーム情報
- `選手ID/選手名/背番号`: 該当選手の情報
- `スターティングフラグ`: スターターか否かのフラグ（1=スターター、0=ベンチ）
- `プレイタイム`: 該当選手のプレイタイム（「DNP」=出場なし）

#### 3.2 得点関連（約10項目）
- `得点`: **該当選手の得点数** ← 得点数の可視化
- `3P成功数/3P試投数`: **該当選手の3Pシュート成功数/試投数** ← 3Pシュートの可視化
- `2P成功数/2P試投数`: **該当選手の2Pシュート成功数/試投数** ← 2Pシュートの可視化
- `ダンク`: **該当選手のダンクシュート成功数（試投数はなし）** ← ダンクの可視化、インパクトのあるコンテンツ
- `フリースロー成功数/フリースロー試投数`: 該当選手のフリースロー成功数/試投数

#### 3.3 その他の統計（約10項目）
- `ファウル/被ファウル`: 該当選手のファウル数/被ファウル数
- `オフェンスリバウンド/ディフェンスリバウンド/トータルリバウンド`: **該当選手のリバウンド数** ← リバウンドの可視化
- `ターンオーバ`: 該当選手のターンオーバ数（シュート以外の行為でボールコントロールを失った場合）
- `アシスト`: **該当選手のアシスト数** ← アシストの可視化
- `スティール`: **該当選手のスティール数** ← スティールの可視化
- `ブロックショット/被ブロックショット`: **該当選手のブロックショット数/被ブロックショット数** ← ブロックショットの可視化

#### 3.4 特殊統計（約5項目）
- `ファストブレイクポイント`: **該当選手のファストブレイク（速攻）からのポイント数** ← 速攻の可視化
- `2Pインサイドポイント`: 該当選手のペイントエリア内からのポイント数
- `セカンドチャンスポイント`: 該当選手の所属チームがオフェンスリバウンドを取り、再度のオフェンスにて該当選手が記録したポイント数

### 4. チーム成績データ（試合毎）

選手成績データ（試合毎）と同様の項目がチーム単位で記録。

**追加項目**:
- `ポイントフロムターンオーバー`: 該当チームが相手チームのターンオーバー後のオフェンスにて記録したポイント数

### 5. 選手成績データ（シーズン毎）

#### 5.1 基本情報（約15項目）
- `カップID/カップ名`: 該当のシーズンのID・名称
- `カンファレンス`: 地区を表すコード値（1=東地区、2=西地区、3=中地区）
- `年度`: 該当の年度
- `チームID/チーム名`: 該当選手所属チームの情報
- `選手ID/選手名`: 該当選手の情報
- `ポジション名`: 該当選手のポジション名
- `在籍フラグ`: 該当選手がチームに所属しているかのフラグ（0=退団・移籍済み、1=在籍）
- `試合数`: 該当選手の出場試合数

#### 5.2 得点関連（約20項目）
- `総得点/平均得点`: **該当選手の総得点/1試合平均得点** ← 総得点/平均得点の可視化
- `成功率FG/成功FG/試投FG`: 該当選手のフィールドゴール成功率/成功数/試投数
- `平均3P/成功率3P/成功3P/試投3P`: **該当選手の1試合平均3P成功数/成功率/成功数/試投数** ← 3Pの可視化
- `平均2P/成功率2P/成功2P/試投2P`: **該当選手の1試合平均2P成功数/成功率/成功数/試投数** ← 2Pの可視化
- `平均フリースロー/成功率フリースロー`: 該当選手の1試合平均フリースロー成功数/成功率

#### 5.3 その他の統計（約20項目）
- `平均ファウル/平均被ファウル`: 該当選手の1試合平均ファウル数/被ファウル数
- `平均オフェンスリバウンド/平均ディフェンスリバウンド/平均トータルリバウンド`: **該当選手の1試合平均リバウンド数** ← リバウンドの可視化
- `平均アシスト`: **該当選手の1試合平均アシスト数** ← アシストの可視化
- `平均スティール`: **該当選手の1試合平均スティール数** ← スティールの可視化
- `平均ターンオーバ`: 該当選手の1試合平均ターンオーバー数
- `平均ブロックショット`: **該当選手の1試合平均ブロックショット数** ← ブロックショットの可視化
- `スターティング数`: 該当選手のスターターでの出場試合数
- `プレイタイム秒`: 該当選手の総プレイタイムの秒数
- `EFF`: **該当選手のEFF（総合的な貢献度を測る指標）** ← 総合評価の可視化

### 6. チーム成績データ（シーズン毎）

選手成績データ（シーズン毎）と同様の項目がチーム単位で記録。

**追加項目**:
- `勝数/負数`: 該当チームの勝数/負数
- `総失点/平均失点`: 該当チームの総失点/平均失点
- `ファストブレイク`: 該当チームのファストブレイクポイント（速攻からの得点）
- `インサイド2P`: 該当チームのペイントエリア内での得点
- `セカンドチャンス`: 該当チームのセカンドチャンスポイント（オフェンスリバウンド後の得点）
- `ポイントフロムターンオーバー`: 該当チームのポイントフロムターンオーバー（相手のターンオーバー後の得点）

### 7. 試合情報データ

- `試合ID`: 該当試合ID
- `試合年月日`: 該当試合年月日
- `カップID/カップ名`: 該当試合が開催されたシーズンのID・名称
- `最大ピリオド`: 該当試合が終了したピリオド（クォーター）
- `節`: 該当試合の節
- `ホームチームID/ホームチーム名`: 該当試合のホームチーム情報
- `アウェイチームID/アウェイチーム名`: 該当試合のアウェイチーム情報
- `スタジアムID/スタジアム名`: **該当試合が行われた試合会場のID・名称** ← スタジアム別の観客動員分析
- `開始時刻/終了時刻`: 該当試合の試合開始・終了時刻
- `観客動員数`: **該当試合の観客動員数** ← 観客動員予測、マーケティング施策の効果測定
- `ホーム得点/アウェイ得点`: 該当試合のホーム・アウェイチームの得点
- `主審名/副審名1/副審名2`: 該当試合の審判名

### 8. 選手プロフィールデータ

- `カップID`: 該当のシーズンのID
- `チームID`: 該当選手所属チームのチームID
- `選手ID`: 該当選手のID
- `背番号`: 該当選手の背番号
- `選手名`: **該当選手の選手名** ← 選手の特徴分析、コンテンツ生成
- `ポジション`: 該当選手のポジション
- `国籍`: 該当選手の国籍
- `生年月日`: 該当選手の生年月日
- `身長`: **該当選手の身長** ← 身体能力の可視化
- `体重`: **該当選手の体重** ← 身体能力の可視化
- `在籍フラグ`: 該当選手がチームに所属しているかのフラグ（0=退団・移籍済み、1=在籍）

- スポーツは大きなイベントはテレビで見るが、生の観戦をするほど興味が湧かない

   - 見どころデータを活用したSNS投稿

## 📍 iタウンページデータ

### データの全体像

| 観点       | 標準出力データ     | 統計情報データ    |
| -------- | ----------- | ---------- |
| データ粒度    | 事業所・店舗単位    | 地域 × 業種単位  |
| 1レコードの意味 | 1電話番号（1事業所） | 1住所 × 1業種  |
| 主用途      | 検索・マスタ・名寄せ  | 分析・統計・商圏把握 |
| 更新区分     | 新規／削除／訂正あり  | なし         |
| 個票識別     | 可能          | 不可         |

### 標準出力データ（個票データ）

**データの性格**:
- 電話帳掲載情報を網羅した詳細マスタデータ
- CRM、検索サービス、名寄せ処理の基盤用途を想定
- 月次更新を前提とした差分管理設計

**ファイル仕様**:
- 形式: CSV（可変長、CRLF）
- 文字コード: 1バイト：JIS／2バイト：Shift-JIS
- ファイル名: iTown.txt

**主な項目構成**:
- **識別・管理系**: 通番（8桁）、情報種別（一般／新規／削除／訂正）、ファイル種別
- **電話情報**: 市外局番、電話番号、電番略符号（FAX、代表電話等）
- **名称情報**: カナ名称、漢字名称、法人種類コード
- **住所情報（正規化）**: 住所コード（11桁）、都道府県名、市区郡町村名、大字・通称名、字・丁目名、街区番号・部屋番号、方書
- **分類情報**: 基本分類コード／名（64分類）、TP分類コード／名（業種分類）
- **補助情報**: 郵便番号、置換識別表示、通称識別表示

### 統計情報データ（ピンポイント版）

**データの性格**:
- 個別事業所を含まない集計前提データ
- 地域別・業種別の分布把握を目的
- マーケティング・統計分析向け

**ファイル仕様**:
- レコード長: 固定長 224バイト
- 住所コード: 上位5桁 または 11桁（選択制）

**主な項目構成**:
- **識別・地域**: 通番、住所コード（上位5桁／下位6桁）、通称識別表示
- **分類・件数**: 基本分類コード、TP分類コード、件数（常に「1」固定 → SUMで集計）
- **表示用項目**: 基本分類名、TP分類名、都道府県名、市区郡町村名、大字・通称名、字・丁目名

### 使い分けの指針

| 利用目的      | 推奨データ |
| --------- | ----- |
| 検索サービス構築  | 標準出力  |
| 事業所マスタ整備  | 標準出力  |
| 名寄せ・突合    | 標準出力  |
| 商圏分析      | 統計情報  |
| 業種分布分析    | 統計情報  |
| 地域マーケティング | 統計情報  |

### スポーツデータとの組み合わせ活用例

| 組み合わせ | 活用アイデア |
| --- | --- |
| プロ野球 + 標準出力 | 札幌ドーム周辺の飲食店・宿泊施設マッピング |
| Jリーグ + 統計情報 | サッカー観戦客向けエリアマーケティング |
| Bリーグ + 標準出力 | 試合日の周辺店舗プロモーション企画 |
| 観客動員 + 統計情報 | 試合日の商圏影響分析、経済効果測定 |

**結論**:
- **「誰が・どこに・何をしているか」を扱うなら標準出力**
- **「どこに・どの業種が・どれだけあるか」を扱うなら統計情報**

両者は詳細／簡易の関係ではなく、**目的が明確に分離された補完関係**である。
