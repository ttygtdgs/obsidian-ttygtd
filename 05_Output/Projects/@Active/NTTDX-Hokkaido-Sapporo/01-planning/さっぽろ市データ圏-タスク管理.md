# さっぽろ市データ圏 タスク管理

## 目標：来年2月の施策検討

### 今日のコミット範囲（検討事項）

#### 1. コンテンツの検討
- [ ] イベントのテーマ設定
  - 札幌市の特性を活かしたテーマ（観光、雪まつり、食文化、冬季課題など）
  - データ活用の目的（課題解決、サービス創出、市民参加など）
- [ ] イベント形式の検討
  - アイデアソン vs ハッカソン vs その他の形式
  - オンライン vs オフライン vs ハイブリッド
  - 開催時間・期間の設定
- [ ] 対象参加者の設定
  - 市民、学生、企業、行政職員など
  - 参加者のスキルレベル（初心者歓迎か、技術者向けか）
- [ ] 提供するデータセットの選定
  - さっぽろ市データ圏で公開されているデータ
  - イベント用に新たに公開すべきデータ
  - データの説明資料・サンプルコードの準備

#### 2. 参加インセンティブ
- [ ] 参加者へのインセンティブ設計
  - 賞金・賞品の設定
  - 表彰制度（最優秀賞、優秀賞、特別賞など）
  - 副賞（企業とのマッチング機会、インキュベーションプログラム参加権など）
- [ ] 継続的な参加促進
  - コミュニティ形成への誘導
  - フォローアップイベントの企画
  - 成果物の展示・紹介機会の提供
- [ ] 企業・団体へのインセンティブ
  - スポンサー企業へのメリット（認知度向上、CSR効果など）
  - データ提供企業へのメリット
  - 協力機関（大学、研究機関など）へのメリット

#### 3. 必要データ収集の段取り
- [ ] データ収集の必要性の検討
  - 既存データで十分か
  - 新たなデータ収集が必要か
  - データ収集の目的と優先順位
- [ ] データ収集方法の検討
  - 行政データの追加公開
  - 民間企業からのデータ提供依頼
  - 市民参加型データ収集
  - IoTセンサーなどの活用
- [ ] データ収集のスケジュール
  - データ公開までの期間
  - イベント開催までに必要な準備期間
  - データの品質確保・検証期間
- [ ] データ提供体制の整備
  - データ提供元との調整
  - データフォーマットの統一
  - メタデータの整備
  - APIの提供（必要に応じて）

#### 4. イベント形式の検討（アイデアソン vs ハッカソン）
- [ ] アイデアソンの特徴とメリット
  - 技術的ハードルが低い
  - 幅広い参加者が可能
  - アイデア創出に特化
  - プロトタイプ作成までが中心
- [ ] ハッカソンの特徴とメリット
  - 実際に動くサービス・アプリの開発
  - 技術者向け
  - より実用的な成果物が期待できる
  - 継続的な開発につながりやすい
- [ ] ハイブリッド形式の検討
  - アイデアソン→ハッカソンの2段階形式
  - 初心者向けセッションと技術者向けセッションの併設
  - オンラインとオフラインの併用
- [ ] 札幌市の状況に適した形式の選定
  - 参加者の想定スキルレベル
  - 期待する成果物の種類
  - リソース（予算、人員、会場など）
  - 継続性の観点
