# 札幌市アイデアソン - ペルソナ別データ活用シナリオ検討

## 1. ペルソナ設定（観戦経験×興味軸）

[[札幌市アイデアソン-データセット統合整理]]に基づき、「観戦経験の有無」と「興味の有無」の2軸で4つのペルソナを定義します。

| ペルソナ | 観戦経験 | 興味・関心 | ターゲット層の定義 | 特徴・インサイト |
| :--- | :---: | :---: | :--- | :--- |
| **A. コアファン**<br>（分析・熱狂層） | あり | あり | **熱狂的ファン・経験者**<br>年パス保持者、過去に競技経験がある層 | ・普通の解説では物足りない<br>・「なぜそのプレーが起きたか」を深く知りたい<br>・自分なりの戦術や予想を語りたい |
| **B. ライトファン**<br>（エンタメ・付き添い層） | あり | なし | **付き添い・イベント参加層**<br>誘われて行くが試合自体には飽きがち | ・ルールがあまりわからない<br>・試合時間が長く感じる（退屈な時間がある）<br>・試合以外の「ご飯」や「イベント」を楽しみたい |
| **C. 潜在ファン**<br>（推し活・オンライン層） | なし | あり | **TV・配信観戦・推し活層**<br>現地未経験だがハイライト等は見る | ・現地に行くハードルが高い（ルール、マナー、一人観戦）<br>・特定の選手（推し）だけを追いかけたい<br>・選手の見た目やストーリーに惹かれる |
| **D. 無関心層**<br>（地域住民層） | なし | なし | **地域生活者・一般市民**<br>スポーツに興味がない一般市民 | ・「地元・札幌/北海道」には愛着がある<br>・健康、グルメ、地域貢献には関心がある<br>・スポーツは「自分とは関係ない世界」と感じている |

---

## 2. 各ペルソナごとのデータ活用シナリオ

### A. コアファン（経験あり × 興味あり）

**コンセプト**: **「プロのような眼を持つ」**
詳しいデータを見れるようにして、ファン自身の「分析したい」「解説したい」という気持ちを満たすアイデア。

| 対象競技 | 活用データセット | 具体的な活用シナリオ |
| :--- | :--- | :--- |
| **プロ野球** | ・投球位置/打球位置座標<br>・イベントデータ詳細 | **「次の一球予測 & 配球答え合わせ」**<br>投手の配球（投げたコースの記録）と打者の得意コースを重ねて見て、スマホで「次の一球」を予測するゲーム。結果とすぐに答え合わせができる。 |
| **Jリーグ** | ・トラッキングデータ<br>（XY座標・速度） | **「ボールがない動き評価AI」**<br>ボールを持っていない選手の動き（囮の動き、スペースを作る動き）をデータから見つけ出す。「テレビに映らない貢献」を数値化して褒める仕組み。 |
| **Bリーグ** | ・シュート座標<br>・詳細スタッツ | **「勝負所（クラッチタイム）分析」**<br>第4Q残り5分の勝負所に絞ったシュート成功率や、難しい体勢での成功率を見える化し、本当に「勝負強い選手」を見つける。 |

### B. ライトファン（経験あり × 興味なし）

**コンセプト**: **「退屈させない観戦ナビ」**
試合が動かない時間（退屈な時間）を埋めて、わかりやすい面白さを提供するアイデア。

| 対象競技 | 活用データセット | 具体的な活用シナリオ |
| :--- | :--- | :--- |
| **全競技** | ・観客動員数<br>・試合経過時間 | **「混雑回避＆グルメ満喫アラート」**<br>過去のデータと今の試合状況から、「今ならトイレが空いています」「ハーフタイム直前のグルメ空き時間」を予測してスマホに通知する。 |
| **プロ野球** | ・打球速度/飛距離<br>・投球速度 | **「『音』と『衝撃』の体感ビジュアライザ」**<br>ルールがわからなくても、「時速160km」や「すごい打球速度」をスマホの振動や画面エフェクトで表現し、凄さを直感的に伝える。 |
| **Jリーグ** | ・選手の走行距離<br>・スプリント回数 | **「頑張ってる選手発見機」**<br>戦術はわからなくても、「今一番走っている選手」「一番ダッシュしている選手」をリアルタイムでランキング表示し、応援のきっかけを作る。 |

### C. 潜在ファン（経験なし × 興味あり）

**コンセプト**: **「"推し"から入るスタジアム」**
競技全体ではなく「個人」への興味を入り口に、現地に行くハードルを下げるアイデア。

| 対象競技 | 活用データセット | 具体的な活用シナリオ |
| :--- | :--- | :--- |
| **全競技** | ・選手プロフィール<br>（身長/体重/出身/年齢） | **「自分とそっくりな選手検索」**<br>自分と「身長・体重・年齢・出身地」などが似ている選手を3競技からまとめて検索。「自分と同じ体格のプロ選手」を親近感の入り口にする。 |
| **Bリーグ** | ・イベントデータ<br>（ダンク/3Pシュート） | **「『推し活躍』即時通知」**<br>ダンクや3Pシュートが決まった瞬間、推し登録しているファンのスマホにだけ「○○選手がダンクを決めました！」と特別な通知が届く。アプリを開くとそのプレーのスタッツ詳細が見られる。 |
| **プロ野球** | ・選手プロフィール<br>・年俸データ | **「もしも自分がプロ選手なら？シミュレーター」**<br>同い年の選手の年俸や活躍を見て、架空の「プロ野球選手としての自分」の人生をシミュレーションする育成ゲームのような体験。 |

### D. 無関心層（経験なし × 興味なし）

**コンセプト**: **「生活とつながるスポーツデータ」**
スポーツを「健康」や「地域」の話題に変えて、無関心な人との接点を作るアイデア。

| 対象競技 | 活用データセット | 具体的な活用シナリオ |
| :--- | :--- | :--- |
| **全競技** | ・選手プロフィール<br>（出身地・出身校） | **「ご近所ヒーロー応援団」**<br>「同じ小学校/中学校出身」「実家の近く出身」の選手が出場する時だけ通知が来て、地元の飲食店で使えるクーポンがもらえる仕組み。 |
| **Jリーグ** | ・トラッキングデータ<br>（走行距離/速度） | **「プロに挑戦！健康ウォーキング」**<br>プロ選手の1試合の走行距離（約10-13km）と、市民の1週間の歩数を競争させる。クリアすると観戦チケット割引などがもらえる健康アプリ。 |
| **プロ野球** | ・観客動員数<br>・試合終了時間 | **「混雑予報としてのスポーツ」**<br>ドーム周辺の混雑を避けるためにデータを使う逆転の発想。「今日は試合があるからこの道は避けよう」という情報を、周辺のお店のセール情報（寄り道誘導）と一緒に提供する。 |

## 3. 統合的なアイデアの方向性（競技ごちゃまぜ活用）

3つの競技データが揃っている札幌ならではの**「競技ごちゃまぜ（クロススポーツ）」**のアプローチも有効です。

1.  **「北海道・身体能力図鑑」**
    *   野球の投げる速さ、サッカーの走る速さ・持久力、バスケのジャンプ力・高さ。これらを同じ物差しで比較し、子どもたちが「自分はどのスポーツに向いているか」を発見できる体験型展示。
2.  **「札幌・熱狂ヒートマップ」**
    *   ドーム（野球/サッカー）と北海きたえーる（バスケ）の観客動員と歓声（SNS等の盛り上がり）をマップ上で可視化し、札幌のどこが今「熱い」かをリアルタイムで可視化するシティプロモーション。
