# アイデアソン用データセット情報 - iタウンページ

## 概要

本資料は、札幌市アイデアソンで使用するタウンページデータ（2種）の仕様を整理したものです。

**使用データ**:
1. iタウンページDB 標準出力仕様書（個票データ）
2. iタウンページ 統計情報仕様書・ピンポイント版（集計データ）

## 1. 全体像の整理

| 観点       | 標準出力データ     | 統計情報データ    |
| -------- | ----------- | ---------- |
| データ粒度    | 事業所・店舗単位    | 地域 × 業種単位  |
| 1レコードの意味 | 1電話番号（1事業所） | 1住所 × 1業種  |
| 主用途      | 検索・マスタ・名寄せ  | 分析・統計・商圏把握 |
| 更新区分     | 新規／削除／訂正あり  | なし         |
| 個票識別     | 可能          | 不可         |

## 2. iタウンページDB 標準出力仕様（個票データ）

### 2.1 データの性格

- 電話帳掲載情報を網羅した**詳細マスタデータ**
- CRM、検索サービス、名寄せ処理の基盤用途を想定
- 月次更新を前提とした差分管理設計

### 2.2 ファイル仕様

| 項目    | 内容                      |
| ----- | ----------------------- |
| 形式    | CSV（可変長、CRLF）           |
| 文字コード | 1バイト：JIS／2バイト：Shift-JIS |
| ファイル名 | iTown.txt               |

### 2.3 主な項目構成

#### 識別・管理系

- 通番（8桁）
- 情報種別（一般／新規／削除／訂正）
- ファイル種別

#### 電話情報

- 市外局番
- 電話番号
- 電番略符号（FAX、代表電話等）

#### 名称情報

- カナ名称（最大74桁、最大5分割）
- 漢字名称（最大35桁、最大5分割）
- 法人種類コード

#### 住所情報（正規化）

- 住所コード（11桁）
- 都道府県名
- 市区郡町村名
- 大字・通称名
- 字・丁目名
- 街区番号・部屋番号
- 方書

#### 分類情報

- 基本分類コード／名（64分類）
- TP分類コード／名（業種分類）

#### 補助情報

- 郵便番号
- 置換識別表示（漢字置換の有無・内容）
- 通称識別表示（正規／通称）

### 2.4 設計上の重要点

- **漢字・外字対策が厳密**（類似文字・カナ置換・フリガナ化）
- **住所は完全分解構造**で行政コード連携が容易
- 更新判定ロジックが明文化されており、差分バッチ処理に向く

## 3. iタウンページ 統計情報仕様（ピンポイント版）

### 3.1 データの性格

- 個別事業所を含まない**集計前提データ**
- 地域別・業種別の分布把握を目的
- マーケティング・統計分析向け

### 3.2 ファイル仕様

| 項目    | 内容                |
| ----- | ----------------- |
| レコード長 | 固定長 224バイト        |
| 文字種   | C：1バイト、K：2バイト（漢字） |
| 住所コード | 上位5桁 または 11桁（選択制） |

### 3.3 項目構成

#### 識別・地域

- 通番
- 住所コード（上位5桁／下位6桁）
- 通称識別表示（0：正規、1：通称）

#### 分類・件数

- 基本分類コード
- TP分類コード
- 件数（常に「1」固定）

#### 表示用項目

- 基本分類名
- TP分類名
- 都道府県名
- 市区郡町村名
- 大字・通称名
- 字・丁目名
- 街区〜部屋番号

#### その他

- 未定義領域（スペース）

### 3.4 設計上の重要点

- **件数が1固定**＝後段処理でSUMする前提
- 個人・法人の特定は不可能
- 地域×業種のクロス集計に最適化

## 5. アイデアソンでの活用ポイント

### 標準出力データの活用シーン

- **「誰が・どこに・何をしているか」を扱う場合**
- 特定エリアの飲食店リストアップ
- スポーツ関連施設の位置特定
- 地域密着型店舗のマスタ作成

### 統計情報データの活用シーン

- **「どこに・どの業種が・どれだけあるか」を扱う場合**
- 札幌市内の業種分布ヒートマップ作成
- エリア別の商業集積度分析
- 観光客向け施設の集中エリア特定

### データスタジアムデータとの組み合わせ

| 組み合わせ例 | 活用アイデア |
| --- | --- |
| プロ野球 + 標準出力 | 札幌ドーム周辺の飲食店・宿泊施設マッピング |
| Jリーグ + 統計情報 | サッカー観戦客向けエリアマーケティング |
| Bリーグ + 標準出力 | 試合日の周辺店舗プロモーション企画 |

---

## 6. 結論

- **「誰が・どこに・何をしているか」を扱うなら標準出力**
- **「どこに・どの業種が・どれだけあるか」を扱うなら統計情報**

両者は詳細／簡易の関係ではなく、**目的が明確に分離された補完関係**である。
