# 経営指導員フローとAI活用メモ

## 1. 現状フロー（As-Is）

1. **相談受付**
   - Webページや電話から県窓口へ相談
   - 相談内容はメール・紙・口頭で管理され、検索や共有が難しい
2. **アサイン検討**
   - 担当職員が過去実績や個人の記憶を頼りに経営指導員を決定
   - 判断根拠が残らず、アサインの属人化が発生
3. **支援実施**
   - 指導員が現地／オンラインで相談対応
   - 活動記録は個人のメモ・Excelに散在
4. **報告・評価**
   - 県への報告は紙やExcelで提出
   - 組織的な可視化・振り返りができず、改善サイクルが回らない

## 2. kintone導入後フロー（To-Be）

1. **相談受付 → kintone自動登録**
   - Webフォームの入力内容をそのままkintoneに保存
   - 相談企業属性・課題カテゴリ・希望時期などを構造化
2. **アサインワークフロー**
   - 相談レコードに紐づく形で指導員選定を実施
   - 稼働状況・専門分野をリアルタイムに参照し、決定ログを残す
3. **活動記録**
   - 訪問記録、提案資料、議事録などをkintoneアプリに登録
   - 進捗ステータスや次アクションをタスク管理できる
4. **報告・ダッシュボード**
   - kintone上のデータを元にダッシュボード化
   - 相談件数／対応スピード／成果指標／フォロー漏れアラート等を可視化

## 3. AI化で付加価値を出せるポイント

### 3.1 優先度の高いAI化施策

#### ① 既存情報のDB化
- **目的**：過去の相談記録、提案資料、成功事例などを構造化データベース化
- **内容**：
  - 紙・Excel・メールに散在する過去データをAIで構造化・kintoneへ移行
  - テキスト抽出・分類・タグ付けを自動化
  - 検索可能なナレッジベースを構築
- **効果**：過去の知見を即座に参照可能にし、提案品質の底上げ

#### ② 問い合わせフローのチャットボット構築
- **目的**：相談受付時の前ヒアリングを自動化し、データ品質と初動速度を向上
- **内容**：
  - Web／LINE等で相談者から自動ヒアリング
  - 必須項目はシナリオで担保、自由記述はLLMで要約・タグ付け
  - 取得情報をkintoneへ自動登録
- **効果**：24/7対応、データ品質の平準化、指導員の事前準備時間短縮

#### ③ 提案資料の自動作成支援
- **目的**：過去レコードから類似事例を参照し、提案資料の下書きを自動生成
- **内容**：
  - kintone内の過去レコード（成功事例・提案資料）をRAGで検索
  - 相談内容に基づいて類似事例を抽出・要約
  - 提案資料のテンプレートを自動生成（指導員が編集・完成）
- **効果**：提案作成時間の短縮、過去の成功パターンの活用

### 3.2 その他のAI化アイデア

| 工程 | AI化アイデア | 効果 |
| --- | --- | --- |
| アサイン | **マッチングスコア算出エンジン**（過去実績・専門性・稼働） | 属人化の解消、適切な人材配置 |
| 支援準備 | **指導員向け事前準備支援**（相談要約・類似事例提示・QA） | 事前準備時間の短縮、品質底上げ |
| 活動記録 | **テキスト自動要約**、**TODO抽出** | 記録負荷を削減しながらデータを即時登録 |
| 報告・改善 | **異常検知／アラート**（フォロー漏れ・成果停滞）、**自動レポート生成** | 自律的なモニタリングと改善サイクルの定着 |

## 4. 次アクションメモ

### 優先実装項目
1. **既存情報のDB化**：過去データの構造化・移行（kintone導入と並行）
2. **問い合わせフローのチャットボット構築**：相談受付の自動化
3. **提案資料の自動作成支援**：過去レコード活用による提案支援

### 前提整備
- 相談受付フォームとkintoneの項目設計を先に固める（AI連携の前提）
- データ利活用ポリシー／個人情報の取り扱いルールを整理し、AI運用要件に反映
- 過去データの移行範囲・匿名化ルールを明確化
