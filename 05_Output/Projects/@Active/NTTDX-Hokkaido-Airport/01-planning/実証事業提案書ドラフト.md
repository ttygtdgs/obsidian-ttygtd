# 北海道エアポート様向け
# 空港混雑可視化ソリューション ご提案

---

## 1. ご提案の背景

### 貴社の課題認識
- 発着数50本/hの維持・最大化が収益向上の鍵
- 定刻運航のためには**人の流れの最適化**が必須
- 課題箇所は把握しているが、**データに基づくアクション**が未実施

### 現状の課題
| 課題 | 現状 | 影響 |
|-----|------|------|
| 人流データなし | 課題箇所は把握しているがデータ未取得 | 改善PDCAが回らない |
| 混雑状況把握 | スタッフが巡回して確認 | リアルタイム対応が困難 |
| 乗客への情報提供 | 案内放送のみ | 行動分散が促せない |

---

## 2. ご提案概要

### コンセプト
**「まず見える化」で、改善の第一歩を踏み出す**

既存のSafieカメラを活用し、空港内の混雑状況をリアルタイムで可視化するダッシュボードを構築します。

### 提案ソリューション

## 3. 機能詳細

### 3.1 リアルタイム混雑表示

**機能概要**
- エリア別の混雑度を3段階（低/中/高）で色分け表示
- フロアマップ上にヒートマップで可視化
- 更新間隔：1-5分（調整可能）

**対象エリア（例）**
- 保安検査場
- 出入国審査
- 搭乗口エリア
- 手荷物受取所

### 3.2 混雑推移グラフ

**機能概要**
- 時間帯別の混雑推移を折れ線グラフで表示
- 過去データとの比較表示
- 曜日・フライトスケジュールとの相関分析

### 3.3 アラート通知

**機能概要**
- 混雑度が閾値を超えた際に自動通知
- Slack/LINE/メールなど複数チャネル対応
- 通知先・閾値はカスタマイズ可能

### 3.4 レポート機能

**機能概要**
- 日次/週次/月次レポートの自動生成
- ボトルネック箇所の特定
- 改善施策の効果測定

```
【データソース】
Safie カメラ映像/データ
     │
     ↓
【データ取得】
Safie API / データエクスポート
     │
     ↓
【バックエンド】
クラウドサーバー（AWS/GCP）
- データ蓄積
- 混雑度算出
- アラート判定
     │
     ↓
【フロントエンド】
Webダッシュボード
- PC/タブレット対応
- レスポンシブデザイン
```
