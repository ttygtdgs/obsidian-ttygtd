# 🎯 北海道エアポート案件 - 提案方針

## 💡 基本アプローチ

### コンセプト
**「まず見える化、すぐ動くプロトタイプで価値を証明」**

### 我々の強み
- 生成AI + バイブコーディングによる**高速プロトタイピング**
- 既存サービス・APIを組み合わせた**素早い価値提供**
- 「動くもの」で**顧客の意思決定を支援**

---

## 📦 提案メニュー

### 提案①: 混雑状況リアルタイムダッシュボード

**概要**
Safieカメラの映像/データを活用し、空港内の混雑状況をリアルタイムで可視化するWebダッシュボードを開発。

**技術スタック**
```
Safie API/データ連携
  ↓
Next.js / React ダッシュボード
  ↓
混雑度ヒートマップ + グラフ表示
```

**できること**
- エリア別の混雑度を色分け表示
- 時間帯別の混雑推移グラフ
- アラート機能（閾値超えで通知）

**提供価値**
- スタッフが一目で混雑状況を把握
- ボトルネック箇所のデータ蓄積
- 改善施策の効果測定基盤

**開発期間**: 2-3週間
**想定工数**: 1-2人月

---

### 提案②: 混雑予測・アラートシステム

**概要**
過去データ + フライトスケジュールから混雑を予測し、事前にスタッフへアラート通知するシステム。

**技術スタック**
```
過去の混雑データ + フライト情報
  ↓
シンプルな予測ロジック（時系列分析）
  ↓
Slack/LINE/Web通知
```

**できること**
- 15-30分後の混雑予測表示
- 混雑予測時のアラート自動送信
- 予測精度のフィードバック収集

**提供価値**
- 事前対応で混雑を未然に防止
- スタッフ配置の最適化
- 予測→実績のPDCAサイクル構築

**開発期間**: 3-4週間
**想定工数**: 2-3人月

---

### 提案③: 乗客向け混雑情報アプリ / サイネージ連携

**概要**
乗客がスマホで保安検査・出入国の待ち時間を確認できるWebアプリ。サイネージへの表示も可能。

**技術スタック**
```
混雑データ（提案①と連携）
  ↓
PWA / Webアプリ
  ↓
QRコードでアクセス / サイネージ表示
```

**できること**
- 現在の待ち時間目安を表示
- 混雑ピークの予告
- 多言語対応（日/英/中/韓）

**提供価値**
- 乗客の行動分散を促進
- 顧客満足度向上
- インバウンド対応強化

**開発期間**: 2-3週間
**想定工数**: 1-2人月

---

### 提案④: カメラ設置場所検討支援ツール

**概要**
空港フロアマップ上でカメラ設置候補をプロットし、カバレッジや死角を可視化するツール。

**技術スタック**
```
空港フロアマップ（画像/CAD）
  ↓
Webアプリでインタラクティブ編集
  ↓
カバレッジ可視化 + レポート出力
```

**できること**
- 設置候補のドラッグ&ドロップ配置
- カメラ画角のシミュレーション
- 監視/マーケティング両観点でのスコアリング

**提供価値**
- 設置場所検討の効率化
- 関係者間の合意形成支援
- 増設計画の可視化

**開発期間**: 2週間
**想定工数**: 1人月

## ✅ なぜこのアプローチか

### 1. スモールスタート
- 小さく始めて、価値を証明してから拡大
- リスクを抑えつつ、成果を見せる

### 2. 素早いプロトタイプ
- バイブコーディングで2-3週間で「動くもの」を提供
- 顧客と一緒に磨き上げる

### 3. 既存インフラ活用
- Safie（既に導入済み）のデータを活用
- 新規インフラ投資なしで開始可能

### 4. 段階的拡張
- Phase 1の成果を基に、Phase 2以降を提案
- 顧客との信頼関係を構築
