# WeightCheer - 技術スタック

## 技術選定

### フロントエンド（iOS）

| 項目 | 選定 | 理由 |
|------|------|------|
| 言語 | Swift | Apple公式、型安全 |
| UIフレームワーク | SwiftUI | 宣言的UI、最新のiOS開発標準 |
| 最低対応OS | iOS 17 | SwiftUIの最新機能活用、ユーザー層的にも十分 |
| アーキテクチャ | MVVM | SwiftUIとの親和性が高い |
| グラフ描画 | Swift Charts | Apple純正、SwiftUI統合済み |

### バックエンド

| 項目 | 選定 | 理由 |
|------|------|------|
| BaaS | Firebase | 個人開発で手軽、無料枠が十分 |
| 認証 | Firebase Authentication | Apple Sign In対応 |
| データベース | Cloud Firestore | リアルタイム同期、オフライン対応 |
| プッシュ通知 | Firebase Cloud Messaging (FCM) | Firebaseとの統合が容易 |
| ストレージ | Firebase Storage | プロフィール画像用 |

### 選定理由：Firebase vs CloudKit

| 観点 | Firebase | CloudKit |
|------|----------|----------|
| リアルタイム同期 | ◎ 標準対応 | △ 実装コスト高 |
| グループ共有 | ◎ 柔軟 | △ 制約が多い |
| プッシュ通知 | ◎ FCM統合 | ○ APNs直接 |
| 無料枠 | ◎ Spark plan十分 | ◎ 無料 |
| 学習コスト | ○ ドキュメント豊富 | △ 情報少なめ |
| Android展開 | ◎ 対応可 | × 不可 |

→ **グループ機能・リアルタイム同期・将来のAndroid展開を考慮しFirebaseを選定**

## データモデル設計

### Firestore コレクション構成

```
users/
  {userId}/
    - name: string
    - iconUrl: string?
    - createdAt: timestamp

groups/
  {groupId}/
    - name: string
    - ownerId: string (記録者のuserId)
    - targetWeight: number?
    - inviteCode: string
    - createdAt: timestamp

    members/ (サブコレクション)
      {userId}/
        - role: "recorder" | "supporter"
        - joinedAt: timestamp

    records/ (サブコレクション)
      {recordId}/
        - weight: number
        - memo: string?
        - date: string (YYYY-MM-DD)
        - createdAt: timestamp

        comments/ (サブコレクション)
          {commentId}/
            - userId: string
            - userName: string
            - text: string
            - type: "text" | "stamp"
            - stampId: string? (スタンプの場合)
            - createdAt: timestamp
```

### セキュリティルール方針

- `users/{userId}`: 本人のみ読み書き可
- `groups/{groupId}`: メンバーのみ読み取り可、オーナーのみ書き込み可
- `groups/{groupId}/records`: メンバー全員読み取り可、記録者のみ書き込み可
- `groups/{groupId}/records/{recordId}/comments`: メンバー全員読み書き可、削除は投稿者のみ

## コスト見積もり（Firebase Spark無料枠）

| リソース | 無料枠 | 想定使用量 | 判定 |
|----------|--------|-----------|------|
| Firestore読み取り | 50,000回/日 | ~1,000回/日 | ◎ |
| Firestore書き込み | 20,000回/日 | ~100回/日 | ◎ |
| Storage | 5GB | ~100MB | ◎ |
| Authentication | 無制限 | 少数 | ◎ |

→ **個人開発・小規模利用であれば無料枠内で運用可能**
