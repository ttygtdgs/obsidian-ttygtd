# コンテンツピラー設計

## 概要

オウンドメディア全体のコンテンツを支える4つの柱（ピラー）を定義する。各ピラーはターゲットの課題に直結し、4ピラーを組み合わせることで「DX/AI実務の専門家」「実践者としての信頼性」「独立起業家としての共感」「知的生産のプロフェッショナル」を立体的に表現する。

---

## 4ピラー全体像

```
┌─────────────────────────────────────────────┐
│            オウンドメディア                    │
│         「DX/AIの実践知を届ける」              │
├───────────┬───────────┬───────────┬──────────┤
│ ピラー1   │ ピラー2   │ ピラー3   │ ピラー4  │
│ DX実践知  │ AI活用実務│ 起業      │ ナレッジ │
│           │           │ ジャーニー│ 管理     │
│ 40%       │ 30%       │ 15%       │ 15%      │
└───────────┴───────────┴───────────┴──────────┘
```

---

## ピラー1: DX実践知（配分: 40%）

### テーマ
地方自治体・中堅企業のDX事例、データ活用の実践、組織変革のリアル。

### ターゲット
- 企業のDX推進担当者（30-50代）
- 地方自治体のデジタル推進課職員
- IT企業の研修企画担当

### 主要キーワード
- 地方自治体DX、データ活用、データ基盤、DX推進
- デジタル人材育成、組織変革、業務改善
- Looker Studio、GA4、データ分析
- 中小企業DX、DXコンサルティング

### コンテンツ例（5本）

| # | タイトル案 | チャネル | 形式 |
|---|-----------|---------|------|
| 1 | 地方自治体DX - データ活用で住民サービスが変わった3つの事例 | ブログ | 長文記事 |
| 2 | 中堅企業のデータ基盤構築 - 半年で成果を出すロードマップ | ブログ → LinkedIn | ガイド記事 |
| 3 | DX人材育成プログラムの設計方法 - 研修講師の視点から | YouTube | ミニ講座 |
| 4 | Looker Studioで作るKPIダッシュボード実践ガイド | YouTube → ブログ | チュートリアル |
| 5 | 「DXが進まない」組織に共通する3つの構造的問題 | note → LinkedIn | 考察記事 |

### 既存資産との連携
- [[05_Output/Projects/@Active/NTTDX-Hokkaido-Sapporo/]] の講義素材を再利用
- [[05_Output/Projects/@Active/NTTDX-Customer-Expansion-Mission/]] の分析手法を一般化
- [[04_Memory/Business/Marketing/content-marketing-strategy|コンテンツマーケティング戦略]] の「現状→障壁→未来像」フレームワーク

## ピラー2: AI活用実務（配分: 30%）

### テーマ
生成AI（Claude, ChatGPT）、AI開発ツール（Cursor, Windsurf）の実践的な活用法。表面的なレビューではなく、業務で本当に使える知見。

### ターゲット
- AI活用に意欲的なエンジニア（20-40代）
- AIを業務に取り入れたいビジネスパーソン
- 個人クリエイター・フリーランス

### 主要キーワード
- Claude活用法、生成AI実務、プロンプトエンジニアリング
- Cursor使い方、AIコーディング、AI開発ツール
- AI業務効率化、AIワークフロー
- Dify、AIエージェント、RAG

### コンテンツ例（5本）

| # | タイトル案 | チャネル | 形式 |
|---|-----------|---------|------|
| 1 | Cursor完全ガイド - AI駆動開発で生産性を3倍にする方法 | YouTube → ブログ | チュートリアル |
| 2 | Claude活用術 - プロンプトの書き方で結果が10倍変わる | ブログ → X | 実践ガイド |
| 3 | 生成AIツール比較2026 - 業務別おすすめの使い分け | ブログ | 比較記事 |
| 4 | AIエージェント入門 - Difyで業務自動化フローを作る | YouTube | 実演動画 |
| 5 | 「AIに仕事を奪われる」は本当か - 構造的に考える | note → LinkedIn | 考察記事 |

### 既存資産との連携
- [[05_Output/Areas/Content-Creation/YouTube-Channel/00-channel-strategy|YouTubeチャンネル戦略]] のCursor Tips シリーズを継承

## ピラー3: 起業ジャーニー（配分: 15%）

### テーマ
法人化プロセス、独立のリアル、フリーランスから起業家への転身。成功だけでなく葛藤や試行錯誤も含めた等身大の記録。

### ターゲット
- 独立・起業を検討しているエンジニア/コンサルタント
- 副業から本業への移行を考えている人
- 同世代の起業家・フリーランス

### 主要キーワード
- エンジニア起業、フリーランス法人化、一人法人
- 独立準備、起業リアル、副業から独立
- 事業計画、収益モデル、営業戦略
- ワークライフバランス、家族と起業

### コンテンツ例（5本）

| # | タイトル案 | チャネル | 形式 |
|---|-----------|---------|------|
| 1 | エンジニアが法人設立してみた - 手続きと費用のリアル | note → ブログ | 体験記 |
| 2 | フリーランスから法人化を決めた3つの理由 | note → LinkedIn | エッセイ |
| 3 | 一人法人の営業戦略 - 案件獲得のリアルな方法 | ブログ → メルマガ | 実践ガイド |
| 4 | 起業1年目の収支を全公開 - 予想と現実のギャップ | note | 体験記 |
| 5 | 家族を持つエンジニアの独立 - 妻への説明から始めた話 | note → X | エッセイ |

### 既存資産との連携
- 法人設立プロジェクトの進捗を素材として活用
- 事業計画策定プロセスをリアルタイムで共有

## ピラー4: ナレッジ管理（配分: 15%）

### テーマ
Obsidian、Second Brain、知的生産術。情報を知識に変え、アウトプットにつなげる方法論。

### ターゲット
- 情報整理に悩むビジネスパーソン
- Obsidian/Notionに興味がある知識労働者
- PKM（Personal Knowledge Management）愛好者

### 主要キーワード
- Obsidian使い方、セカンドブレイン、Zettelkasten
- ナレッジ管理、知的生産術、PKM
- PARA法、メモ術、情報整理
- Obsidianプラグイン、Obsidianワークフロー

## ピラー間のクロスオーバー

4ピラーは独立ではなく、掛け合わせることで独自性が生まれる。

| 掛け合わせ | コンテンツ例 |
|-----------|------------|
| DX実践知 x AI活用 | 「自治体DXにおける生成AI活用 - 実プロジェクトの知見」 |
| AI活用 x ナレッジ管理 | 「Claude + Obsidianで知識管理を自動化する方法」 |
| 起業ジャーニー x DX実践知 | 「DXコンサルタントとして独立する - 市場と案件の見つけ方」 |
| ナレッジ管理 x 起業 | 「起業家のための知的生産システム - Obsidianで事業計画を管理する」 |
| AI活用 x 起業 | 「一人法人にAIは何ができるか - 業務自動化の実践」 |
