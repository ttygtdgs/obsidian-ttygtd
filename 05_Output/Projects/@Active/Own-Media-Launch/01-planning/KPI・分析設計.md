# KPI・分析設計

## 概要

オウンドメディア6チャネルのKPI設計と分析体制を定義する。数値だけを追うのではなく、「なぜその数値を見るのか」を明確にし、月次レビューで継続的に改善する。

---

## KPI設計の基本方針

### 指標の3層構造

```
┌─────────────────────────────────┐
│  ビジネス目標（North Star）     │  → 月間問い合わせ件数
├─────────────────────────────────┤
│  チャネルKPI（各チャネル）      │  → PV、登録者、フォロワー等
├─────────────────────────────────┤
│  行動指標（運用品質）           │  → 投稿頻度、制作時間等
└─────────────────────────────────┘
```

### North Star Metric
**月間問い合わせ件数**（コンサルティング・研修案件）

すべてのチャネルKPIは最終的にこの指標に貢献しているかで評価する。

| 平均滞在時間 | 2分 | 3分 | 3分30秒 | GA4 |

## 分析ツール設定ガイド

### Google Analytics 4 (GA4)

**設定手順**:
1. GA4プロパティ作成
2. Next.jsにGA4タグ埋め込み（`@next/third-parties/google`）
3. コンバージョンイベント設定:
   - `newsletter_signup` — メルマガ登録
   - `contact_click` — 問い合わせボタンクリック
   - `cta_click` — CTA遷移
4. カスタムディメンション設定:
   - `content_pillar` — ピラー別分析用
   - `content_type` — コンテンツ種別

**主要レポート**:
- トラフィック獲得レポート（流入元分析）
- ページ別レポート（人気コンテンツ分析）
- コンバージョンレポート（CTA効果分析）
- ユーザー属性レポート（ターゲット確認）

### Google Search Console

**設定手順**:
1. ドメイン所有権の確認
2. サイトマップ送信（`/sitemap.xml`）
3. Next.jsの`generateSitemaps()`で自動生成

**主要レポート**:
- 検索パフォーマンス（クエリ別の表示回数・クリック数・順位）
- インデックス登録状況
- Core Web Vitals
- モバイルユーザビリティ

### YouTube Analytics

**主要レポート**:
- リーチ（インプレッション、CTR、トラフィックソース）
- エンゲージメント（視聴時間、平均視聴維持率）
- オーディエンス（登録者の推移、視聴者層）
- 収益（将来的にモネタイズ開始時）

## 月次レビュー設計

### レビュー実施ルール
- **実施日**: 毎月第1月曜日
- **所要時間**: 60分
- **記録先**: `02-materials/月次レビュー-YYYY-MM.md`

## 四半期レビュー設計

### レビュー項目

1. **戦略適合性**: ピラー配分は適切か、ターゲットは正しいか
2. **チャネルROI**: 時間投資に見合うリターンがあるか
3. **コンテンツ循環**: 循環モデルは機能しているか
4. **技術スタック**: ツールの追加・変更が必要か
5. **KPI見直し**: 目標値は現実的か、指標は適切か
